# 013_v2.3: Fix Pending Clips Appearing as Separate Recordings

**Date:** December 27, 2025
**Version:** 2.3 (Fixes v2.2 regression - clips appearing separately on home screen)
**Issue:** Multiple pending clips create separate "Recording" files instead of grouping under one parent
**Time:** 15-20 minutes
**Complexity:** MEDIUM (requires understanding parent-child relationship)

---

## 🐛 Bug Discovery (Post v2.2)

**What v2.2 fixed:**
- ✅ Each pending clip now has unique ID (no more duplicate key errors)
- ✅ All three clips (001, 002, 003) visible in record screen

**What v2.2 broke:**
- ❌ Each pending clip creates a separate "Recording XX" on home screen
- ❌ Instead of one "Recording 01" containing 3 pending clips, we get 3 separate recordings
- ❌ When you navigate back, only the clicked recording shows (loses siblings)

**Screenshots Evidence:**
1. Record screen: Shows "Clip 001", "Clip 002", "Clip 003" together ✅
2. Home screen: Shows "Recording 01", "Recording 02", "Recording 03" separately ❌
3. Click "Recording 03": Only shows "Clip 003" (001 and 002 disappeared) ❌

---

## 🔍 Root Cause Analysis

**The Architectural Confusion:**

We're mixing two concepts:
1. **Recording File** = Parent container (appears on home screen)
2. **Pending Clip** = Child of a recording (appears within record screen)

**What v2.2 did wrong:**

```typescript
// EVERY offline recording creates a NEW Recording file:
const nextNumber = getNextRecordingNumber(getClips());  // ← 01, 02, 03
const newClip = createNewClip('', nextNumber, '');      // ← "Recording 01", "Recording 02", "Recording 03"
```

Result:
```
clips[] = [
  { id: "clip-ABC", title: "Recording 01", pendingClipTitle: "Clip 001", status: "pending" },
  { id: "clip-DEF", title: "Recording 02", pendingClipTitle: "Clip 002", status: "pending" },
  { id: "clip-GHI", title: "Recording 03", pendingClipTitle: "Clip 003", status: "pending" }
]
```

All three appear as separate recordings on home screen!

---

## ✅ The Correct Architecture

**What we actually want:**

```
clips[] = [
  {
    id: "clip-ABC",           // ← PARENT recording file
    title: "Recording 01",
    status: "pending",         // ← Shows "waiting" on home screen
    pendingClipTitle: null     // ← Not a pending clip itself, it's the parent
  },
  {
    id: "clip-DEF",            // ← CHILD pending clip 1
    title: "Recording 01",     // ← SAME title as parent (for grouping)
    status: "pending-child",   // ← Different status (filtered from home screen)
    pendingClipTitle: "Clip 001",
    parentId: "clip-ABC"       // ← Links to parent
  },
  {
    id: "clip-GHI",            // ← CHILD pending clip 2
    title: "Recording 01",     // ← SAME title as parent
    status: "pending-child",
    pendingClipTitle: "Clip 002",
    parentId: "clip-ABC"
  }
]
```

**Home screen filter:**
```typescript
// Only show non-child clips
const homeScreenClips = clips.filter(c => c.status !== 'pending-child');
// Result: Only "Recording 01" shows on home screen
```

**Record screen logic:**
```typescript
// When clicking "Recording 01", load all its children
const parentClip = clips.find(c => c.id === clickedId);
const childClips = clips.filter(c => c.parentId === clickedId);
setSelectedPendingClips(childClips);  // Shows 001 and 002
```

## 🏗️ The Fix: Hybrid Parent-Child Approach

### Change 1: First Recording Creates Parent (KEEP currentClipId)

**File:** `ClipMasterScreen.tsx`
**Lines:** ~1262-1279

#### BEFORE (v2.2 - Creates new recording every time):

```typescript
if (audioId) {
  // PHASE 2.2 FIX: ALWAYS create a NEW clip for each offline recording
  const nextNumber = getNextRecordingNumber(getClips());
  const newClip = createNewClip('', nextNumber, '');
  const clipIdToUpdate = newClip.id;

  log.info('Created new clip for offline recording', {
    clipId: newClip.id,
    title: nextNumber,
    isSuccessive: selectedPendingClips.length > 0
  });

  {
    log.info('Status transition', {
      clipId: clipIdToUpdate,
      from: null,
      to: 'pending',
      trigger: 'offline-save'
    });
```

#### AFTER (v2.3 - Parent/child distinction):

```typescript
if (audioId) {
  let clipIdToUpdate;
  let isChildClip = false;

  // PHASE 2.3: First recording creates PARENT, subsequent recordings create CHILDREN
  if (!currentClipId || selectedPendingClips.length === 0) {
    // Case 1: NO parent exists yet → Create parent Recording file
    const nextNumber = getNextRecordingNumber(getClips());
    const newClip = createNewClip('', nextNumber, '');
    clipIdToUpdate = newClip.id;
    setCurrentClipId(newClip.id);  // ← SAVE parent ID for subsequent recordings

    log.info('Created PARENT recording for offline session', {
      clipId: newClip.id,
      title: nextNumber
    });
  } else {
    // Case 2: Parent exists → Create CHILD pending clip linked to parent
    const childId = `clip-${Date.now()}-${Math.random()}`;
    const parentClip = getClips().find(c => c.id === currentClipId);

    // Create child clip with parent's title (for grouping)
    const childClip = {
      id: childId,
      title: parentClip?.title || 'Recording',  // ← Same title as parent
      date: new Date().toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' }),
      content: '',
      rawText: '',
      formattedText: '',
      status: 'pending-child',  // ← Special status (filtered from home screen)
      parentId: currentClipId,   // ← Links to parent
      audioId: audioId,
      duration: formatDuration(duration),
      pendingClipTitle: getNextPendingClipNumber()
    };

    // Add to storage
    const currentClips = getClips();
    const updatedClips = [...currentClips, childClip];
    sessionStorage.setItem('clips', JSON.stringify(updatedClips));

    clipIdToUpdate = childId;
    isChildClip = true;

    log.info('Created CHILD pending clip', {
      clipId: childId,
      parentId: currentClipId,
      pendingClipTitle: childClip.pendingClipTitle
    });
  }

  // Continue with existing logic for updating status...
  {
    log.info('Status transition', {
      clipId: clipIdToUpdate,
      from: null,
      to: isChildClip ? 'pending-child' : 'pending',
      trigger: 'offline-save'
    });
```

**Changes:**
1. ✅ Added: `if (!currentClipId || selectedPendingClips.length === 0)` - First recording creates parent
2. ✅ Added: `setCurrentClipId(newClip.id)` - Saves parent ID for subsequent recordings
3. ✅ Added: `else` block - Subsequent recordings create child clips
4. ✅ Added: `status: 'pending-child'` - Special status for filtering
5. ✅ Added: `parentId: currentClipId` - Links child to parent
6. ✅ Added: Child clips share parent's `title` - For grouping

### Change 3: Load All Children When Clicking Parent

**File:** `ClipMasterScreen.tsx`
**Function:** `handleClipClick`
**Lines:** ~236-257

#### BEFORE (v2.2 - Only loads clicked clip):

```typescript
else {
  // Pending clip (no content yet)
  setSelectedClip(null);
  setCurrentClipId(clip.id);

  const pendingStatus = clip.status === 'transcribing' ? 'transcribing' : 'waiting';

  setSelectedPendingClips([{
    id: clip.id,
    title: clip.pendingClipTitle || 'Clip 001',
    time: clip.duration || '0:00',
    status: pendingStatus,
    isActiveRequest: isActiveRequest
  }]);  // ← Only ONE clip
```

#### AFTER (v2.3 - Loads parent + all children):

```typescript
else {
  // Pending clip (no content yet) - load parent and all children
  setSelectedClip(null);
  setCurrentClipId(clip.id);

  // Find all child clips belonging to this parent
  const allClips = getClips();
  const children = allClips.filter(c => c.parentId === clip.id);

  // Convert parent + children to PendingClip array
  const pendingClips: PendingClip[] = [];

  // Add children first (they have pendingClipTitle)
  children.forEach(child => {
    const status = child.status === 'transcribing' ? 'transcribing' : 'waiting';
    pendingClips.push({
      id: child.id,
      title: child.pendingClipTitle || 'Clip',
      time: child.duration || '0:00',
      status: status,
      isActiveRequest: child.status === 'transcribing'
    });
  });

  // If parent itself has audio (first recording), include it
  if (clip.audioId) {
    const status = clip.status === 'transcribing' ? 'transcribing' : 'waiting';
    pendingClips.push({
      id: clip.id,
      title: clip.pendingClipTitle || 'Clip 001',
      time: clip.duration || '0:00',
      status: status,
      isActiveRequest: clip.status === 'transcribing'
    });
  }

  setSelectedPendingClips(pendingClips);  // ← ALL pending clips
  setAnimationVariant('fade');
  setRecordNavState('record');
  setActiveScreen('record');
}
```

**Changes:**
1. ✅ Finds all child clips: `allClips.filter(c => c.parentId === clip.id)`
2. ✅ Loads ALL pending clips (parent + children)
3. ✅ Navigating back now shows all 3 clips (001, 002, 003)

## 🔍 Why This Fix Works

**Before (v2.2):**
```
Home Screen:
├── Recording 01 (Clip 001)  ← Separate
├── Recording 02 (Clip 002)  ← Separate
└── Recording 03 (Clip 003)  ← Separate
```

**After (v2.3):**
```
Home Screen:
└── Recording 01 (parent)
    └── When clicked:
        ├── Clip 001 (parent itself)
        ├── Clip 002 (child)
        └── Clip 003 (child)
```

**Key Differences:**

1. **First recording:** Creates parent + sets `currentClipId` (keeps context)
2. **Subsequent recordings:** Create children linked to parent
3. **Home screen:** Filters out children (only shows parents)
4. **Record screen:** Loads parent + all children (shows all pending clips)
5. **Transcription:** Updates all children, merges into parent

## 💡 Why We Don't Need Explicit parentRecordingId Everywhere

**The Question:** "Should every pending clip have an explicit parentRecordingId field?"

**The Answer:** We DO use it, but ONLY for child clips!

**Why this works:**

1. **Parent clip:** Has no `parentId` (it IS the parent)
   - Appears on home screen
   - Represents the recording session

2. **Child clips:** Have `parentId` pointing to parent
   - Hidden from home screen (status='pending-child')
   - Loaded together when parent is clicked

3. **Transcription:** Works by clip ID
   - Each clip (parent + children) transcribes independently
   - No risk of mixing (each has unique ID + audioId)

**Benefits:**
- ✅ Simple to implement (no complex filtering everywhere)
- ✅ Clear parent-child relationship (explicit `parentId`)
- ✅ Home screen stays clean (one recording per session)
- ✅ Record screen shows all related clips (grouped by parentId)
