## Part 2: Fix TypeScript Errors in useOfflineRecording.ts

### Fix #1: Line 85 - Remove Invalid 'pending' Status Check

**Location**: [useOfflineRecording.ts:85](useOfflineRecording.ts#L85)

**Issue**: Comparing to `'pending'` which doesn't exist in ClipStatus type

**FIND**:
```typescript
} else if (currentClip.status === 'pending' || currentClip.status === 'pending-child') {
```

**REPLACE WITH**:
```typescript
} else if (currentClip.status === 'pending-child') {
```

**Why**: The new ClipStatus type doesn't include `'pending'`. Parents have `status: null`, children have `status: 'pending-child'`. The `'pending'` check is legacy code that doesn't apply anymore.

## Part 3: Fix TypeScript Configuration (Map Iteration Errors)

### Fix #6: Update TypeScript Target to ES2015

**Location**: [tsconfig.json:4](tsconfig.json#L4)

**Issue**: `target: "es5"` is too old (from 2009) - can't iterate Map/Set objects, causes 3 TypeScript errors

**FIND**:
```json
{
  "compilerOptions": {
    "target": "es5",
```

**REPLACE WITH**:
```json
{
  "compilerOptions": {
    "target": "es2015",
```

**Why This Fixes the Errors**:
- ES2015 introduced native iterator protocol
- `Map.prototype.entries()` returns iterator that can be used in `for...of` loops
- All modern browsers support ES2015 (released 2015, it's 2026 now!)
- Your app already requires ES2015 (React, Next.js, modern features)

**Errors Fixed**:
- ✅ Line 1222: `Type 'MapIterator<[string, Clip[]]>' can only be iterated...`
- ✅ Line 1230: `Parameter 'a' implicitly has an 'any' type`
- ✅ Line 1230: `Parameter 'b' implicitly has an 'any' type`

**Browser Compatibility** (all safe):
- ✅ Chrome 51+ (2016)
- ✅ Firefox 54+ (2017)
- ✅ Safari 10+ (2016)
- ✅ Edge 15+ (2017)
