## Bug #1: Text Appending Broken (Only Last Clip Shows)

### Symptom
User recorded 2 clips offline:
- Clip 001: "This is clip 001 recording in 01 file"
- Clip 002: "This is clip 002 recording in 01 file"

After transcription, final content only has:
```
"This is clip zero zero two inside file recording zero zero one."
```

**❌ Clip 001's text is MISSING!**

### Root Cause (Line 598-611)

```typescript
// Update parent (safe - verified parent exists)
const updatedContent = {
  rawText: currentParent.rawText
    ? currentParent.rawText + '\n\n' + result.text
    : result.text,
  formattedText: currentParent.formattedText
    ? currentParent.formattedText + ' ' + formattedText
    : formattedText,
  content: currentParent.formattedText  // ⚠️ PROBLEM: STALE VALUE
    ? currentParent.formattedText + ' ' + formattedText
    : formattedText,
};

updateClip(currentParent.id, updatedContent);
```

**The Issue**: `currentParent` is captured at line 472:
```typescript
const { clips: allClips, updateClip, deleteClip, getClipById } = useClipStore.getState();
```

This is a **snapshot** taken at the START of `processAllPendingClips`.

**What happens**:
1. **Clip 001 processes**: 
   - `currentParent.formattedText` = "" (empty)
   - Appends: "" + "Clip 001 text" = "Clip 001 text"
   - Updates parent ✅

2. **Clip 002 processes** (SAME LOOP ITERATION):
   - `currentParent.formattedText` = "" ❌ **STILL EMPTY!** (stale snapshot)
   - Appends: "" + "Clip 002 text" = "Clip 002 text"
   - **OVERWRITES** parent with just Clip 002 text ❌

**Result**: Only last clip's text survives!

---

### The Fix

**Need to refetch parent FRESH before appending**:

```typescript
// ❌ BROKEN (Line 598-611):
const updatedContent = {
  content: currentParent.formattedText  // ← STALE
    ? currentParent.formattedText + ' ' + formattedText
    : formattedText,
};

// ✅ FIXED:
// Refetch parent to get latest content
const freshParent = getClipById(currentParent.id);  // ← FRESH DATA
const updatedContent = {
  rawText: freshParent.rawText
    ? freshParent.rawText + '\n\n' + result.text
    : result.text,
  formattedText: freshParent.formattedText
    ? freshParent.formattedText + ' ' + formattedText
    : formattedText,
  content: freshParent.formattedText  // ← FRESH VALUE
    ? freshParent.formattedText + ' ' + formattedText
    : formattedText,
};
```

## Bug #2: Pending Clips UI Disappears After Second Recording

### Symptom
1. Record Clip 001 offline → Shows in UI ✅
2. Press Record again for Clip 002
3. After Done → **UI shows BLANK** (no pending clips visible) ❌
4. Navigate to home and back → Both clips appear ✅

### Root Cause (Line 394-405)

```typescript
// ✅ NEW Case 2.5: Recording from pending clip (no content yet, but has audioId)
else if (activeScreen === 'record' && selectedPendingClips.length > 0) {
  // Keep currentClipId - we're adding to the SAME clip file
  // This allows multiple pending recordings in one file
  setIsAppendMode(true);
  setCurrentClipId(selectedPendingClips[0].id);  // ⚠️ PROBLEM: Sets to CHILD ID
  setAppendBaseContent('');
  log.debug('Recording from pending clip (adding successive recording)', {
    clipId: selectedPendingClips[0].id,
    pendingTitle: selectedPendingClips[0].title
  });
  setTimeout(() => startRecordingHook(), 200);
}
```

**The Issue**: Line 398 sets `currentClipId` to the CHILD clip's ID (`selectedPendingClips[0].id`), not the PARENT.

**What happens**:
1. User records Clip 001
   - Parent created with ID: `clip-1768050175182-y8nptd3n3`
   - Child 001 created with ID: `clip-1768050175180-xyz123`
   - `currentClipId` = parent ID ✅

2. User presses Record again (line 394 condition passes)
   - Line 398: `setCurrentClipId(selectedPendingClips[0].id)`
   - `currentClipId` = child 001 ID ❌ (should be parent ID)

3. User records and presses Done
   - `handleOfflineRecording` receives `currentClipId` = child 001 ID
   - Tries to append to child (line 75 in useOfflineRecording.ts)
   - Logic breaks because child has `status: 'pending-child'` (not null)

4. selectedPendingClips selector (line 168-196):
   ```typescript
   const children = clips.filter(c => c.parentId === currentClipId);
   ```
   - Filters by `currentClipId` (which is now child 001 ID)
   - No clips have child 001 as their parent
   - Returns empty array
   - **UI shows blank** ❌

---

### The Fix

**Don't change currentClipId when recording from pending clip**:

```typescript
// ❌ BROKEN (Line 394-405):
else if (activeScreen === 'record' && selectedPendingClips.length > 0) {
  setIsAppendMode(true);
  setCurrentClipId(selectedPendingClips[0].id);  // ← Sets to child ID (WRONG)
  setAppendBaseContent('');
  setTimeout(() => startRecordingHook(), 200);
}

// ✅ FIXED:
else if (activeScreen === 'record' && selectedPendingClips.length > 0) {
  setIsAppendMode(true);
  // Don't change currentClipId - it's already set to parent ID
  // selectedPendingClips[0].parentId === currentClipId already
  setAppendBaseContent('');
  setTimeout(() => startRecordingHook(), 200);
}
```

**Or even better, explicitly use parent ID**:

```typescript
else if (activeScreen === 'record' && selectedPendingClips.length > 0) {
  setIsAppendMode(true);
  // Ensure currentClipId is the parent (not child)
  const parentId = selectedPendingClips[0].parentId || selectedPendingClips[0].id;
  if (currentClipId !== parentId) {
    setCurrentClipId(parentId);  // ← Sets to PARENT ID (correct)
  }
  setAppendBaseContent('');
  setTimeout(() => startRecordingHook(), 200);
}
```
