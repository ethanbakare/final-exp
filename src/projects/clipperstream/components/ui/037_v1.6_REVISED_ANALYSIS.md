## Why 037_v1.6's Root Cause Analysis IS Correct

### The Problem Flow (Confirmed by Debug Evidence)

1. **Recording stops** → Blob created with `type: 'audio/webm;codecs=opus'` ✅
2. **Blob stored in IndexedDB** → Type logged correctly ✅  
3. **Blob retrieved from IndexedDB** → Type is lost (empty string) ❌
4. **Blob sent to API** → API receives blob with no MIME type → **500 error** ❌

### This IS an IndexedDB Round-Trip Issue

Your colleague's analysis in `037_v1.6` was **correct**:

> "IndexedDB sometimes loses Blob.type property, causing API to reject audio"

This is a **known browser bug**, especially in Safari and some versions of Chrome. The Blob object is serialized/deserialized during the IndexedDB round-trip, and the `type` property gets lost in the process.

**Status**: ❌ **CRITICAL - WILL CAUSE TYPESCRIPT ERROR**

The plan adds `mimeType: 'audio/webm;codecs=opus'` to child clips (lines 53, 94), but the `Clip` interface in `clipStore.ts` does NOT have a `mimeType` field.

**This WILL fail with TypeScript error:**

**Modify `audioStorage.ts`** to store MIME type separately and recreate blob on retrieval:

- ✅ Fixes the issue for ALL callers (not just auto-retry)
- ✅ No need to thread MIME type through call chains
- ✅ No need to store MIME type in clips
- ✅ Centralizes the fix in one place

**Implementation**:

**Part 1** - Update `storeAudio` (line 66-70):
```typescript

  id: audioId,

  mimeType: blob.type,  // ← Store separately

```

**Part 2** - Update `getAudio` (line 103-108):
```typescript
if (result) {
  // Recreate blob with correct MIME type if lost
  const mimeType = result.mimeType || 'audio/webm';
  const correctedBlob = result.blob.type === ''
    ? new Blob([result.blob], { type: mimeType })
    : result.blob;
  
  log.debug('Audio retrieved from IndexedDB', {
    audioId,
    size: correctedBlob.size,
    originalType: result.blob.type,
    storedMimeType: result.mimeType,
    finalType: correctedBlob.type,
    wasRecreated: result.blob.type === ''
  });
  
  resolve(correctedBlob);
}
```
