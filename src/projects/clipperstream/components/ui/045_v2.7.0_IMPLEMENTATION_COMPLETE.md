## Key Implementation Details

### No Clipboard Copy During Auto-Retry
Per user requirement, `formatChildTranscription` does NOT call `navigator.clipboard.writeText`:
- Document is not focused during background auto-retry
- Clipboard API requires user interaction/focus
- Would throw `NotAllowedError` and mask real bugs

### Fire-and-Forget Title Generation
Title generates after **first clip** completes (not after all clips):
- Matches online behavior (title from first clip)
- Doesn't block remaining clip processing
- Robust for edge case: user goes offline after first clip

### Spacing Strategy
Uses single space `' '` for both raw and formatted text:
- Consistent with current append mode implementation
- AI formatter decides paragraph breaks
- Context helps AI make smart decisions

### Orphaned Children Cleanup
If parent deleted while children pending:
- Detect parent missing in `processParentChildren`
- Delete all orphaned children
- Delete associated audio blobs
- Prevents data leaks and inconsistent state

## Architecture Flow

```
Go Online
  ↓
Find Pending Children (status='pending-child')
  ↓
Group by Parent ID
  ↓
Sort Children by Creation Time (oldest first)
  ↓
For Each Parent:
  ├─ Process First Child
  │   ├─ Transcribe (no context)
  │   ├─ Format (no context)
  │   ├─ Merge to Parent → Show Immediately
  │   ├─ Delete Child
  │   └─ Generate Title (fire-and-forget)
  │
  └─ Process Remaining Children (if any)
      ├─ Accumulate Batch in Memory
      │   ├─ Transcribe with context
      │   ├─ Format with cumulative context
      │   └─ Delete child after processing
      │
      └─ Merge Batch to Parent → Show Once
```
