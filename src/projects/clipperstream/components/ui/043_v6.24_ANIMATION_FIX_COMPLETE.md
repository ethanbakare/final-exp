## 🎯 Summary

Fixed two critical text animation bugs where existing text would incorrectly animate when:
1. **Scenario 1**: User records live → Goes offline → Records new pending clip → Existing text re-animates ❌
2. **Scenario 2**: User records offline → Goes to home → Comes online (background transcription) → Navigates into file → Text animates ❌

**Root Cause**: Animation logic used component-level ref (`prevContentLengthRef`) which resets on unmount and doesn't track user presence.

**Solution**: Track animation state in Zustand + check if user is viewing when text appears.

## 📝 Technical Notes

### Why `undefined` instead of `false`?

```typescript
hasAnimated: userIsViewing ? undefined : true
```

**Reason**: 
- `undefined` = "not set yet" → Component will animate, then set to `true`
- `false` would mean "explicitly set to false" → Component might interpret differently
- `true` = "already animated" → Skip animation

### Why 100ms Delay in useEffect?

```typescript
setTimeout(() => {
  updateClip(selectedClip.id, { hasAnimated: true });
}, 100);
```

**Reason**: Ensures animation starts before marking as complete. Without delay, might mark before CSS animation kicks in.
