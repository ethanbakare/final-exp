## 🐛 CRITICAL BUGS FROM v2.5.2

### Bug #1: Content Bleeding Between Clips (REGRESSION)

**What the user saw:**
- Click Recording 01 (parent with pending children) → Shows "This is clip 006 in recording 02"
- This is transcribed content from Recording 02 showing in Recording 01's view
- Click back to home, then click Recording 01 again → Content disappears (correct behavior)

**Root Cause:**
```typescript
// ClipMasterScreen.tsx handleClipClick line 234-254
if (clip.content) {
  // Transcribed clip - show content
  setSelectedClip(clip);
  setContentBlocks([{
    id: clip.id,
    text: clip.content,  // Set contentBlocks to this clip's content
    animate: false
  }]);
  setRecordNavState('complete');
  setActiveScreen('record');
} else {
  // Parent clip with no content (pending children)
  setSelectedClip(null);
  setRecordNavState('record');
  setActiveScreen('record');
  // ❌ BUG: contentBlocks is NOT cleared here!
}
```

**Why it happens:**
1. User views Recording 02 (has content) → `contentBlocks = [{ text: "This is clip 006..." }]`
2. User clicks back to home
3. User clicks Recording 01 (parent with NO content, has pending children)
4. `handleClipClick` runs the else branch (no content)
5. `setSelectedClip(null)` but `contentBlocks` still has Recording 02's content!
6. ClipRecordScreen renders:
```typescript
// ClipRecordScreen.tsx line 92-95
if (!selectedClip) {
  // No clip selected, show contentBlocks
  return contentBlocks;  // ← Shows stale content from Recording 02!
}
```

**Why it disappears on second click:**
- When you click back to home, `handleBackClick` clears contentBlocks (line 267)
- So second click into Recording 01 has clean state

**This is the SAME bug from v2.3** that we fixed before but it regressed!

### Bug #2: Spinner Not Spinning During Auto-Retry

**What the user saw:**
- Go online → Auto-retry starts
- First clip shows spinning icon for ~1 second
- Then all clips show static "Transcribing..." (no spinning)
- All parent clips stuck with static transcribing icon

**Root Cause - Asynchronous Processing vs Synchronous Loop:**

Looking at console logs from 013_console_2.5.2_debug.md lines 100-149:
```
100: Found pending clips for auto-retry { count: 6 }
101: Status transition { clipId: "child-1", from: "pending-child", to: "transcribing" }
106: Auto-retrying transcription { clipId: "child-1" }
108: Transcription successful { clipId: "child-1" }
109: Status transition { clipId: "child-2", from: "pending-child", to: "transcribing" }  ← Clip 2 starts BEFORE clip 1 finishes processing
113: Background transcription completed { clipId: "child-1" }  ← Clip 1 still processing
114: Starting background formatting { clipId: "child-1" }
116: Auto-retrying transcription { clipId: "child-2" }  ← Clip 2 HTTP starts
118: Transcription successful { clipId: "child-2" }
119: Status transition { clipId: "child-3", from: "pending-child", to: "transcribing" }  ← Clip 3 starts BEFORE clip 1 OR clip 2 finish formatting
126: Text formatted successfully { clipId: "child-1" }  ← Clip 1 FINALLY completes
130-135: Audio deleted, status cleared for clip 1
136: Transcription successful { clipId: "child-3" }  ← Clip 3 HTTP completes
...
```

**The problem:**

The auto-retry loop is SYNCHRONOUS (sequential), but transcription processing is ASYNCHRONOUS (parallel):

```typescript
// ClipMasterScreen.tsx auto-retry loop line 528-587for (const clip of pendingClips) {
  try {
    setActiveTranscriptionClipId(clip.id);  // ← Set immediately

    updateClipById(clip.id, { status: 'transcribing' });
    refreshClips();  // ← React re-renders, parent shows "Transcribing..."

    const audioBlob = await getAudio(clip.audioId!);
    await transcribeRecording(audioBlob);  // ← HTTP completes, returns
    // ⚠️ At this point:
    // - Transcription text is set
    // - isActiveRequest is STILL TRUE (not reset yet)
    // - useTranscriptionHandler will process in a separate useEffect

  } finally {
    setActiveTranscriptionClipId(null);  // ← Clear immediately!
    // ⚠️ This clears BEFORE processing completes!
  }
}
```

**Timeline showing the race condition:**

```
Time 0ms:    Clip 1 starts
             - setActiveTranscriptionClipId(clip1.id)             - isActiveRequest = false (initial)
             - updateClipById(clip1.id, { status: 'transcribing' })
             - refreshClips() → React renders with clip1 status='transcribing'
             - At this render: activeTranscriptionClipId=clip1.id, isActiveRequest=false
             - Parent shows "Transcribing..." (static, no spinner yet)

Time 50ms:   transcribeRecording starts HTTP
             - isActiveRequest = true (set in useClipRecording line 291)
             - React renders with isActiveRequest=true
             - Parent should show spinner BUT might miss this render

Time 500ms:  Clip 1 HTTP completes
             - await transcribeRecording returns
             - isActiveRequest STILL TRUE (not reset in finally block!)
             - transcription text is set

Time 501ms:  Auto-retry loop continues
             - finally block: setActiveTranscriptionClipId(null)
             - Loop moves to clip 2
             - setActiveTranscriptionClipId(clip2.id)
             - updateClipById(clip2.id, { status: 'transcribing' })
             - refreshClips() → React renders
             - At this render: activeTranscriptionClipId=clip2.id, isActiveRequest=true (from clip1)
             - ⚠️ Clip 2 might show spinner briefly (using clip1's isActiveRequest)

Time 520ms:  useTranscriptionHandler processes clip 1
             - formatTranscriptionInBackground() called
             - This is async, takes time...

Time 550ms:  Clip 2 starts HTTP
             - isActiveRequest = true (already true, no change)
Time 800ms:  Clip 1 formatting completes
             - resetRecording() called
             - isActiveRequest = false
             - React renders with isActiveRequest=false             - ⚠️ Clip 2 is transcribing but spinner disappears!

Time 1000ms: Clip 2 HTTP completes
             - await transcribeRecording returns
             - setActiveTranscriptionClipId(null)
             - Loop moves to clip 3
             - setActiveTranscriptionClipId(clip3.id)

Time 1200ms: useTranscriptionHandler processes clip 2
             - resetRecording() → isActiveRequest = false
             - Clip 3 transcription starts but isActiveRequest=false
             - ⚠️ All clips show static "Transcribing..."```

**Why first clip spins briefly:**
- Clip 1 has a brief window (~50-500ms) where `activeTranscriptionClipId=clip1.id` AND `isActiveRequest=true`
- During this window, renders show the spinner
- But once clip 2 starts, the timing becomes misaligned

**Why everything becomes static:**
- After a few clips, `resetRecording()` from previous clips keeps setting `isActiveRequest=false`
- New clips start transcribing but their `isActiveRequest=true` comes later
- The timing windows no longer overlap, so no spinners show

### Fix #2: Track Parent ID Instead of Child ID for Spinner

**Problem with current approach:**
- `activeTranscriptionClipId` tracks which CHILD is being transcribed
- But we clear it immediately after HTTP completes, before processing finishes
- This creates timing issues with `isActiveRequest`

**Better approach:**
- Track which PARENT should show the spinner
- Don't clear until the full processing completes OR until we move to a different parent
- This handles multiple children from same parent gracefully

**Implementation:**

**Step 1: Change state variable**

**File:** [ClipMasterScreen.tsx:137-138](ClipMasterScreen.tsx#L137-L138)

**BEFORE:**
```typescript
// v2.5.2 FIX: Track which clip is currently being transcribed for isActiveRequest derivation
const [activeTranscriptionClipId, setActiveTranscriptionClipId] = useState<string | null>(null);
```

**AFTER:**
```typescript
// v2.5.3 FIX: Track which PARENT should show spinner (not which child)
// This prevents timing issues when children complete processing asynchronously
const [activeTranscriptionParentId, setActiveTranscriptionParentId] = useState<string | null>(null);
```

**Step 2: Update auto-retry loop**

**File:** [ClipMasterScreen.tsx:528-587](ClipMasterScreen.tsx#L528-L587)

**BEFORE:**
```typescript
for (const clip of pendingClips) {
  try {
    // v2.5.2 FIX: Track which clip is being transcribed
    setActiveTranscriptionClipId(clip.id);

    // Update CLIP status only, not nav state
    // ...

    await transcribeRecording(audioBlob);

  } finally {
    // v2.5.2 FIX: Always clear active transcription tracking

  }

```

**AFTER:**
```typescript
for (const clip of pendingClips) {
  try {
    // v2.5.3 FIX: Track which PARENT should show spinner
    const parentId = clip.parentId || clip.id;  // Use parentId if exists, else clip itself    setActiveTranscriptionParentId(parentId);

    // Update CLIP status only, not nav state
    log.info('Status transition', {
      clipId: clip.id,
      from: clip.status,
      to: 'transcribing',
      trigger: 'auto-retry-online'
    });

    updateClipById(clip.id, {
      status: 'transcribing',
      transcriptionError: undefined
    });
    refreshClips();

    // DON'T set currentClipId, isAppendMode, or recordNavState
    // These belong to the current user session, not background retries

    // Fetch audio from IndexedDB
    const audioBlob = await getAudio(clip.audioId!);
    if (!audioBlob) {
      throw new Error('Audio not found in IndexedDB');
    }

    log.debug('Auto-retrying transcription', {
      clipId: clip.id,
      audioId: clip.audioId,
      size: audioBlob.size
    });

    // Transcribe using retrieved blob
    await transcribeRecording(audioBlob);

    // If successful, transcription will be handled by existing useEffect

  } catch (error) {
    log.error('Auto-retry failed for clip', {
      clipId: clip.id,
      error
    });

    // Mark as failed
    log.info('Status transition', {
      clipId: clip.id,
      from: 'transcribing',
      to: 'failed',
      trigger: 'auto-retry-error'
    });

    updateClipById(clip.id, {
      status: 'failed',
      transcriptionError: error instanceof Error ? error.message : 'Auto-retry failed'
    });
    refreshClips();
  }
  // ✅ DON'T clear activeTranscriptionParentId here
  // Let it carry forward to next clip (might be same parent)
}

// ✅ Clear after ALL clips in loop complete
setActiveTranscriptionParentId(null);
```

**Why this works:**
- While transcribing children of parent A, `activeTranscriptionParentId = A` throughout
- Parent A shows spinner continuously while ANY of its children are being transcribed
- When moving to children of parent B, `activeTranscriptionParentId = B`, spinner moves to parent B
- Only cleared after ALL clips in the queue are done
- No timing race conditions because we don't clear between clips

const getDisplayClip = useCallback((
  clip: Clip,
  allClips: Clip[],
  activeTranscriptionParentId: string | null,  // ✅ NEW: parent ID
  isActiveRequest: boolean
): Clip => {
  // Find children for this parent
  const children = allClips.filter(c => c.parentId === clip.id);

  if (children.length === 0) {
    // No children - show clip as-is
    return clip;
  }

  // Derive status from children's states
  const hasTranscribingChildren = children.some(c => c.status === 'transcribing');
  const hasPendingChildren = children.some(c => c.status === 'pending-child');

  let derivedStatus: Clip['status'] = clip.status;

  if (hasTranscribingChildren) {
    // At least one child is transcribing
    derivedStatus = 'transcribing';
  } else if (hasPendingChildren) {
    // Children waiting to transcribe
    derivedStatus = 'pending';
  }

  // v2.5.3 FIX: Derive isActiveRequest by checking if THIS parent is the active one
  // Much simpler: just check parent ID match + isActiveRequest flag
  const derivedIsActiveRequest =
    activeTranscriptionParentId !== null &&
    clip.id === activeTranscriptionParentId &&  // ✅ Direct parent ID comparison
    isActiveRequest;

  return {
    ...clip,
    status: derivedStatus,
    isActiveRequest: derivedIsActiveRequest
  };
}, []);
```

### Fix #2: Parent-Based Spinner Tracking

**Before (child-based tracking with race conditions):**
```
Auto-retry: [clip1, clip2, clip3, clip4] (all children of parent A)

Time 0ms:    activeTranscriptionClipId = clip1.id
             - Parent A checks: children.some(c => c.id === 'clip1') → TRUE
             - isActiveRequest = false (not started yet)
             - Spinner: NO

Time 50ms:   HTTP starts, isActiveRequest = true
             - Spinner: YES (brief window)

Time 500ms:  HTTP completes, await returns
             - activeTranscriptionClipId = null  ← Cleared immediately!
             - Loop moves to clip2
             - activeTranscriptionClipId = clip2.id
             - isActiveRequest = true (from clip1, not reset yet)
             - Parent A checks: children.some(c => c.id === 'clip2') → TRUE
             - Spinner: YES (using clip1's isActiveRequest)

Time 800ms:  clip1 processing completes, resetRecording()
             - isActiveRequest = false  ← Reset
             - clip2 is transcribing but spinner disappears!
             - Spinner: NO  ❌

...continues with misaligned timing
```

**After (parent-based tracking, no race conditions):**
```
Auto-retry: [clip1, clip2, clip3, clip4] (all children of parent A)Time 0ms:    activeTranscriptionParentId = parentA.id
             - Parent A checks: clip.id === parentA.id → TRUE
             - isActiveRequest = false (not started yet)
             - Spinner: NO

Time 50ms:   HTTP starts, isActiveRequest = true
             - Spinner: YES ✅

Time 500ms:  HTTP completes, await returns
             - activeTranscriptionParentId STILL = parentA.id  ← NOT cleared!
             - Loop moves to clip2 (same parent)
             - activeTranscriptionParentId STILL = parentA.id  ← Same parent, no change

Time 800ms:  clip1 processing completes, resetRecording()
             - isActiveRequest = false
             - But clip2 already started HTTP, sets isActiveRequest = true again
             - Spinner: Continues ✅

Time 1200ms: clip2 HTTP starts
             - activeTranscriptionParentId STILL = parentA.id
             - isActiveRequest = true             - Spinner: YES ✅

...continues with spinner active throughout

Time 5000ms: All clips complete, exit loop
             - activeTranscriptionParentId = null
             - Spinner: NO (correct, all done)
```

**Key difference:**
- OLD: Cleared child ID after each clip → timing race conditions
- NEW: Keep parent ID throughout all its children → continuous spinner

## 📊 IMPACT ANALYSIS

### What This Fixes:

1. ✅ **Content bleeding eliminated** - No more stale content from other clips
2. ✅ **Spinner timing fixed** - Spinner shows continuously during each parent's transcription
3. ✅ **Simpler state tracking** - Parent ID instead of child ID (fewer edge cases)
4. ✅ **Handles multiple parents** - Spinner moves correctly between different parents

### Why Parent ID is Better Than Child ID:

**Old approach (child ID):**
- ❌ Must check if any child matches the active child ID
- ❌ Cleared immediately after each clip → timing issues
- ❌ Doesn't handle "same parent, different children" well
- ❌ Race conditions with async processing

**New approach (parent ID):**
- ✅ Direct parent ID comparison (simpler)
- ✅ Persists across all children of same parent
- ✅ Only cleared after entire parent's children complete
- ✅ No race conditions (parent ID stable throughout)
