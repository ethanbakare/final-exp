## Why Previous Implementation Failed

**User said**: "I tried your implementation and had to reverse it."

### Root Cause: Stale Closure

**Without `currentClipId` in dependency array**:

1. Component renders with `currentClipId = 'clip-123'`
2. `processAllPendingClips` callback captures this value
3. User navigates to different clip → `currentClipId = 'clip-456'`  
4. Auto-retry calls the OLD callback with OLD `currentClipId = 'clip-123'`
5. Condition `if (currentClipId === currentParent.id)` uses stale value
6. Even if current parent IS `'clip-456'`, check fails because callback sees `'clip-123'`
7. Nav bar never updates

**Fix**: Add `currentClipId` to dependency array → Callback always has fresh value


## Addressing User's Multi-Clip Concern

**User's question**: "We don't want buttons showing when a DIFFERENT clip finishes transcribing in a different file."

### How the Guard Works

```typescript
if (currentClipId === currentParent.id) {
  setRecordNavState('complete');
}
```

### Example Scenario

```
You're viewing: Parent A (currentClipId = 'A')

Background Process:
  Parent B → Clip 001 transcribes
    → currentParent.id = 'B'
    → Condition: 'A' === 'B'? NO
    → Nav bar NOT updated ✅ CORRECT

  Parent A → Clip 001 transcribes
    → currentParent.id = 'A'
    → Condition: 'A' === 'A'? YES
    → Nav bar updated ✅ CORRECT
```

**Perfect isolation**: Only updates when viewing the specific parent that finished.
