## React Reactivity 101

### How React Should Work

```typescript
// Parent component
const [navState, setNavState] = useState('record');

// Child component
<NavBar navState={navState} />

// When parent calls setNavState('complete'):
// 1. Parent re-renders
// 2. Child receives new navState prop
// 3. Child re-renders with new state
```

### When It Breaks

**Possible Causes**:

1. **Stale Closure**: Function has old reference to state
2. **Missing Dependency**: useCallback/useMemo missing deps
3. **Async Update**: State updates after component unmounted
4. **Prop Not Passed**: NavBar not receiving updated prop
5. **Key Stability**: Component key changed, breaking reference

## Alternative Explanations

### User's Concern: "Different clips in different parent files"

> "we don't want situation whereby it's transcribing a different clip somewhere else inside another file, and it's showing the buttons when the pending clip is still there"

**Analysis**:
This is handled by the condition:
```typescript
if (currentClipId === currentParent.id) {
  setRecordNavState('complete');
}
```

**How it works**:
- `currentClipId`: The clip you're currently viewing
- `currentParent.id`: The clip being transcribed
- If they DON'T match → No nav bar update (correct!)
- If they DO match → Update nav bar (correct!)

**Example Scenario**:
```
Parent File A (currentClipId = 'A')
  - Clip 001 (transcribing)

Parent File B
  - Clip 001 (transcribing)

You're viewing Parent A
Clip in Parent B completes
  → currentClipId ('A') !== currentParent.id ('B')
  → No nav bar update ✅
  
Clip in Parent A completes
  → currentClipId ('A') === currentParent.id ('A')
  → Nav bar updates ✅
```
