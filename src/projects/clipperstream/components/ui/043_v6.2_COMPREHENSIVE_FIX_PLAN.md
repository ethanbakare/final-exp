## ЁЯПЧя╕П Architecture Review: Why One Works, One Doesn't

### The Anti-Pattern (ClipHomeScreen - BEFORE fix)

```typescript
// тЭМ PARENT: Pre-filters data
const homeScreenClips = clips.filter(clip => !clip.parentId);

// тЭМ PARENT: Passes filtered data
<ClipHomeScreen clips={homeScreenClips} />

// тЭМ CHILD: Can't access related entities

```

**Problems**:
1. Parent makes assumptions about child's data needs
2. Child can't derive relationships (children filtered out)
3. Violates single responsibility (parent controls child's data)
4. Hidden dependency (child assumes complete data)

---

### The Correct Pattern (ClipRecordScreen - ALREADY correct)

```typescript
// тЬЕ PARENT: Passes complete data
<ClipRecordScreen pendingClips={getDisplayPendingClips()} />

// тЬЕ getDisplayPendingClips uses full clips array
const selectedPendingClips = useMemo(() => {
  const children = clips.filter(c => c.parentId === currentClipId);  // тЬЕ Has access
  return children.map(...);  // тЬЕ Can derive status
}, [clips, currentClipId]);
```

**Benefits**:
1. Component has full data access for derivation
2. Filtering happens where needed (inside component)
3. Single responsibility (component owns its data requirements)
4. No hidden dependencies

## ЁЯОУ Why This Bug Happened

### Timeline of Introduction

1. **Original design**: ClipMasterScreen passed all clips to ClipHomeScreen
2. **v2.5.2 "optimization"**: Added `homeScreenClips` memo at line 1582
3. **Intent**: "Avoid passing children to home screen (they're not displayed)"
4. **Mistake**: Didn't realize `getDisplayClip` **needs** children to derive parent status

### The Misleading Comment

**Lines 1580-1582:**
```typescript
// PHASE 2.3.1: Filter child clips from home screen display
// Only show parent clips on home screen (children are shown when clicking parent)
```

**Why it's misleading**:
- тЬЕ Correct: "Only show parent clips on home screen"
- тЭМ Wrong assumption: "Therefore, we should filter children before passing data"
- тЭМ Missed requirement: `getDisplayClip` needs children to derive parent status

**The correct interpretation should have been**:
- тЬЕ "Only **DISPLAY** parent clips on home screen" тЖТ Filter in `filteredClips` (line 200)
- тЬЕ "But **PASS** all clips for derivation" тЖТ Pass `clips` to ClipHomeScreen
