## 🐛 Bug Reports from Testing

### Bug #1: Recording Numbers Skip (Recording 01 → Recording 05)

**User report:**
> "I made three clips within my first recording file, then went to an old transcribed file and made another clip there, then made a completely new recording from home. The new recording was named 'Recording 05' instead of 'Recording 02'!"

**Evidence from console (013_console_2.3_debug.md):**
```
Line 48: Created PARENT for offline recording (Recording 01)
Line 61: Created CHILD for offline recording (Clip 002)
Line 74: Created CHILD for offline recording (Clip 003)
Line 89: Created CHILD for offline recording (in transcribed file)
Line 105: Created PARENT for offline recording (Recording 05) ← WRONG!
```

**Storage shows:**
```javascript
clips = [
  { id: "...", title: "Recording 05" },  // ← Should be "Recording 02"
  { id: "...", title: "Recording 04" },  // Ghost file (child)
  { id: "...", title: "Recording 03" },  // Ghost file (child)
  { id: "...", title: "Recording 02" },  // Ghost file (child)
  { id: "...", title: "Recording 01" },  // Real parent
  { id: "...", title: "Mary's Lamb" }    // Transcribed
]
```

**Root cause:**

`getNextRecordingNumber()` counts **ALL clips**, including children!

```typescript
// BUGGY CODE (v2.3.1):
const getNextRecordingNumber = (clips: Clip[]) => {
  const recordingNumbers = clips
    .map(c => c.title.match(/Recording (\d+)/))  // ← Matches CHILDREN too!
    .filter(Boolean)
    .map(match => parseInt(match[1], 10));

  const maxNumber = Math.max(0, ...recordingNumbers);
  return `Recording ${String(maxNumber + 1).padStart(2, '0')}`;
};
```

When you create:
1. Parent "Recording 01" → maxNumber = 1
2. Child (title: "Recording 01") → maxNumber = 1
3. Child (title: "Recording 01") → maxNumber = 1
4. Child in transcribed file (title: "Mary's Lamb") → maxNumber = 1
5. **New parent** → Counts 4 "Recording XX" titles → maxNumber = 4 → Returns "Recording 05" ❌

### Bug #2: Clip Order Changes After Navigation

**User report:**
> "I recorded three clips (001, 002, 003). I exited to home screen, then clicked back into the recording. The clips were now in wrong order: 001, 003, 002!"

**Root cause:**

When loading children via `handleClipClick`, there's no sorting by creation time:

```typescript
// BUGGY CODE (v2.3.1):
const children = clips.filter(c => c.parentId === parentId);
// ← No .sort() → Order is random (depends on storage iteration order)
```

Clip IDs contain timestamps:
- `clip-1766868716300-...` (001) created first
- `clip-1766868723864-...` (002) created second
- `clip-1766868732225-...` (003) created third

But `getClips()` returns them in reverse chronological order (newest first), so children appear backwards.

## ✅ THE FIXES

### Fix #1: Only Count PARENT Recordings

**File:** `ClipMasterScreen.tsx`
**Function:** `getNextRecordingNumber` (near line ~75)

**BEFORE (v2.3.1 - Counts all clips):**

```typescript
const getNextRecordingNumber = useCallback((clips: Clip[]) => {
  const recordingNumbers = clips
    .map(c => c.title.match(/Recording (\d+)/))
    .filter(Boolean)
    .map(match => parseInt(match[1], 10));

  const maxNumber = recordingNumbers.length > 0
    ? Math.max(...recordingNumbers)
    : 0;

  return `Recording ${String(maxNumber + 1).padStart(2, '0')}`;
}, []);
```

**AFTER (v2.3.2 - Only counts parents):**

```typescript
const getNextRecordingNumber = useCallback((clips: Clip[]) => {
  // v2.3.2 FIX: Only count PARENT recordings (exclude children)
  // Children have status='pending-child' or parentId !== undefined
  const parentRecordings = clips.filter(c =>
    c.status !== 'pending-child' &&  // Exclude children
    !c.parentId                      // Exclude anything with a parent
  );

  const recordingNumbers = parentRecordings
    .map(c => c.title.match(/Recording (\d+)/))
    .filter(Boolean)
    .map(match => parseInt(match[1], 10));

  const maxNumber = recordingNumbers.length > 0
    ? Math.max(...recordingNumbers)
    : 0;

  return `Recording ${String(maxNumber + 1).padStart(2, '0')}`;
}, []);
```

**Why this works:**

```
Before fix:
- Parent "Recording 01" → counted ✅
- Child (title: "Recording 01", status: "pending-child") → counted ❌ (should skip)
- Child (title: "Recording 01", status: "pending-child") → counted ❌ (should skip)
- Child (title: "Recording 01", status: "pending-child") → counted ❌ (should skip)
- Result: maxNumber = 4 → "Recording 05" ❌

After fix:
- Parent "Recording 01" → counted ✅
- Child (status: "pending-child") → SKIPPED ✅
- Child (status: "pending-child") → SKIPPED ✅
- Child (status: "pending-child") → SKIPPED ✅
- Result: maxNumber = 1 → "Recording 02" ✅
```

### Fix #2: Sort Children by Creation Time

**File:** `ClipMasterScreen.tsx`
**Function:** `handleClipClick` (line ~247)

**BEFORE (v2.3.1 - No sorting):**

```typescript
const handleClipClick = useCallback((clipId: string) => {
  const clips = getClips();
  const clip = clips.find(c => c.id === clipId);

  if (!clip) return;

  setCurrentClipId(clipId);

  // Load children if this is a parent with pending clips
  const children = clips.filter(c => c.parentId === clipId);

  if (children.length > 0) {
    // Convert children to pending clip format
    const pendingClips = children.map(child => ({
      id: child.id,
      title: child.pendingClipTitle || 'Pending'
    }));

    setSelectedPendingClips(pendingClips);
  } else {
    setSelectedPendingClips([]);
  }

  setActiveScreen('record');
}, [getClips]);
```

**AFTER (v2.3.2 - Sort by timestamp):**

```typescript
const handleClipClick = useCallback((clipId: string) => {
  const clips = getClips();
  const clip = clips.find(c => c.id === clipId);

  if (!clip) return;

  setCurrentClipId(clipId);

  // Load children if this is a parent with pending clips
  const children = clips.filter(c => c.parentId === clipId);

  if (children.length > 0) {
    // v2.3.2 FIX: Sort children by creation time (timestamp in ID)
    // IDs are like "clip-1766868716300-random" where middle part is timestamp
    const sortedChildren = children.sort((a, b) => {
      const timestampA = parseInt(a.id.split('-')[1], 10) || 0;
      const timestampB = parseInt(b.id.split('-')[1], 10) || 0;
      return timestampA - timestampB;  // Ascending (oldest first)
    });

    // Convert children to pending clip format
    const pendingClips = sortedChildren.map(child => ({
      id: child.id,
      title: child.pendingClipTitle || 'Pending'
    }));

    setSelectedPendingClips(pendingClips);

    log.info('Loaded parent with children', {
      parentId: clipId,
      childCount: children.length,
      childOrder: pendingClips.map(p => p.title)  // For debugging
    });
  } else {
    setSelectedPendingClips([]);
  }

  setActiveScreen('record');
}, [getClips]);
```

**Why this works:**

Clip IDs have timestamps:
```
clip-1766868716300-af1yjzwhh  → timestamp: 1766868716300 (Clip 001)
clip-1766868723864-jomnwp3zf  → timestamp: 1766868723864 (Clip 002)
clip-1766868732225-ltg21xgoz  → timestamp: 1766868732225 (Clip 003)
```

Sorting by timestamp ascending gives chronological order:
- 001 (created first)
- 002 (created second)
- 003 (created third)
