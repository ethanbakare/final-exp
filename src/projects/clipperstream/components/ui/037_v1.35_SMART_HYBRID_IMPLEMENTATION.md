## Your Questions Answered

### Q1: Does it fail DeepGram every time before using OpenAI?

**Short answer**: First clip fails, then we remember and skip DeepGram for remaining clips.

**How it works**:
```
Clip 1: Try DeepGram → FAILS (DNS error) → Try OpenAI → SUCCESS
        ↓ (Remember: "DeepGram is down, use OpenAI")
Clip 2: Skip DeepGram → Use OpenAI directly → SUCCESS
Clip 3: Skip DeepGram → Use OpenAI directly → SUCCESS
...
```

**Pattern**: Circuit Breaker (industry standard)

---

### Q2: If retry happens, does it continue with OpenAI?

**Yes** - Once a clip switches to OpenAI, it stays with OpenAI for that clip.

---

### Q3: 5 clips in queue - does each try DeepGram first?

**No** - Smart implementation:
- First clip tries DeepGram → fails → sets "circuit breaker"
- Remaining 4 clips see circuit breaker → use OpenAI directly
- No wasted time

---

### Q4: New recording while queue is processing - parallel or queue?

**PARALLEL** ✅ - New recordings get immediate transcription (separate from queue).

```
Queue processing (background):
├─ Clip 1 (retry) → OpenAI
├─ Clip 2 (retry) → OpenAI
└─ Clip 3 (retry) → OpenAI

User records NEW clip (foreground):
└─ Clip 4 (immediate) → Try DeepGram first (fresh connection might work!)
```

---

### Q5: New online recording uses DeepGram or OpenAI?

**DeepGram first** ✅ - Fresh user action might have better network state.

**Logic**:
- Queue retries (background): Use OpenAI (we know DeepGram failed)
- New recording (foreground): Try DeepGram first (network might have changed)

---

### Q6: When to switch back from OpenAI to DeepGram?

**After queue is empty** ✅ - Reset circuit breaker.

```
Queue finishes → Reset to "DeepGram mode"
Next recording → Try DeepGram again (network might be good now)
```

```typescript
Circuit States:
├─ CLOSED (normal): Try DeepGram
├─ OPEN (failed): Skip DeepGram, use OpenAI
└─ HALF_OPEN (testing): Occasionally try DeepGram to see if it's back
```

### Part 1: Circuit Breaker State Manager

**New file**: `src/projects/clipperstream/api/transcriptionCircuitBreaker.ts`

```typescript
/**
 * Circuit Breaker for Deepgram service
 * Prevents wasting time on repeated DNS failures
 */

type CircuitState = 'CLOSED' | 'OPEN' | 'HALF_OPEN';

interface CircuitBreakerConfig {
  failureThreshold: number;    // Open circuit after N failures
  resetTimeout: number;         // Try again after N milliseconds
  halfOpenAttempts: number;     // Test N times in HALF_OPEN state
}

class TranscriptionCircuitBreaker {
  private state: CircuitState = 'CLOSED';
  private failureCount: number = 0;
  private lastFailureTime: number = 0;
  private config: CircuitBreakerConfig;

  constructor(config: Partial<CircuitBreakerConfig> = {}) {
    this.config = {
      failureThreshold: 1,      // Open after 1 DNS failure (immediate)
      resetTimeout: 60000,      // Try DeepGram again after 60 seconds
      halfOpenAttempts: 1,      // Test once when half-open
      ...config
    };
  }

  /**
   * Check if we should use DeepGram or skip to fallback
   */
  shouldTryDeepgram(isQueueRetry: boolean = false): boolean {
    // For queue retries: if circuit is open, skip DeepGram
    if (isQueueRetry && this.state === 'OPEN') {
      console.log('[CircuitBreaker] OPEN state - skipping DeepGram for queue retry');
      return false;
    }

    // For new recordings: always try DeepGram (fresh connection)
    if (!isQueueRetry && this.state === 'OPEN') {
      console.log('[CircuitBreaker] OPEN state - but trying DeepGram for new recording (fresh connection)');
      return true;
    }

    // Check if we should test DeepGram again (half-open)
    if (this.state === 'OPEN') {
      const timeSinceFailure = Date.now() - this.lastFailureTime;
      if (timeSinceFailure > this.config.resetTimeout) {
        console.log('[CircuitBreaker] Transitioning to HALF_OPEN - testing DeepGram');
        this.state = 'HALF_OPEN';
        return true;
      }
    }

    return true; // CLOSED or HALF_OPEN - try DeepGram
  }

  /**
   * Record successful DeepGram call
   */
  recordSuccess(): void {
    if (this.state !== 'CLOSED') {
      console.log('[CircuitBreaker] DeepGram SUCCESS - closing circuit');
    }
    this.state = 'CLOSED';
    this.failureCount = 0;
    this.lastFailureTime = 0;
  }

  /**
   * Record failed DeepGram call
   */
  recordFailure(error: Error): void {
    // Only open circuit for DNS errors (not other errors like API key)
    const isDNSError = error.message.includes('ENOTFOUND') ||
                       error.message.includes('DNS');

    if (!isDNSError) {
      console.log('[CircuitBreaker] Non-DNS error - not opening circuit');
      return;
    }

    this.failureCount++;
    this.lastFailureTime = Date.now();

    if (this.failureCount >= this.config.failureThreshold) {
      console.log('[CircuitBreaker] Failure threshold reached - OPENING circuit');
      this.state = 'OPEN';
    }
  }

  /**
   * Reset circuit breaker (called when queue is empty)
   */
  reset(): void {
    console.log('[CircuitBreaker] Manual reset - closing circuit');
    this.state = 'CLOSED';
    this.failureCount = 0;
    this.lastFailureTime = 0;
  }

  /**
   * Get current state for debugging
   */
  getState(): { state: CircuitState; failureCount: number; lastFailureTime: number } {
    return {
      state: this.state,
      failureCount: this.failureCount,
      lastFailureTime: this.lastFailureTime
    };
  }
}

// Singleton instance
export const circuitBreaker = new TranscriptionCircuitBreaker();
```

### Part 2: OpenAI Whisper Provider

**New file**: `src/projects/clipperstream/api/whisperProvider.ts`

```typescript
import fs from 'fs';
import OpenAI from 'openai';

export interface TranscriptionResult {
  transcript: string;
  duration: number;
}

const DEBUG = process.env.NODE_ENV === 'development';

function log(...args: any[]) {
  if (DEBUG) console.log(...args);
}

/**
 * Transcribe audio using OpenAI Whisper
 * Used as fallback when Deepgram fails (VPN/DNS issues)
 */
export async function transcribeAudio(
  filePath: string,
  mimeType: string,
  apiKey: string
): Promise<TranscriptionResult> {
  if (!apiKey) {
    throw new Error('OpenAI API key is not configured');
  }

  const openai = new OpenAI({ apiKey });

  log('=== 🎯 OPENAI WHISPER API CALL (FALLBACK) ===');
  log('File path:', filePath);
  log('Using Whisper as DeepGram fallback');

  try {
    // OpenAI expects file stream
    const audioFile = fs.createReadStream(filePath);

    const transcription = await openai.audio.transcriptions.create({
      file: audioFile,
      model: 'whisper-1',
      response_format: 'verbose_json',
    });

    if (!transcription.text || transcription.text.trim() === '') {
      throw new Error('No speech detected in recording. Please try again and speak clearly.');
    }

    log('✅ Whisper transcription successful');

    return {
      transcript: transcription.text.trim(),
      duration: transcription.duration || 0,
    };

  } catch (err) {
    console.error('=== OPENAI WHISPER ERROR ===');
    console.error('Error:', err);
    console.error('============================');

    const errorMessage = err instanceof Error ? err.message : 'Unknown error';
    throw new Error(`Whisper transcription failed: ${errorMessage}`);
  }
}
```

## How It Works in Practice

### Scenario 1: User with VPN, 5 Clips in Queue

**Timeline**:
```
User comes online with VPN
  ↓
Auto-retry starts processing queue

Clip 1 (retry):
  ├─ shouldTryDeepgram(isQueueRetry=true) → YES (circuit CLOSED)
  ├─ Try DeepGram → FAIL (DNS error)
  ├─ circuitBreaker.recordFailure() → Opens circuit
  ├─ Fallback to Whisper → SUCCESS ✅
  └─ Time: ~3 seconds (1s fail + 2s Whisper)

Clip 2 (retry):
  ├─ shouldTryDeepgram(isQueueRetry=true) → NO (circuit OPEN)
  ├─ Skip DeepGram, use Whisper directly
  └─ Time: ~2 seconds (Whisper only)

Clip 3 (retry):
  └─ Same as Clip 2 (Whisper, ~2s)

Clip 4 (retry):
  └─ Same as Clip 2 (Whisper, ~2s)

Clip 5 (retry):
  └─ Same as Clip 2 (Whisper, ~2s)

Queue complete → circuitBreaker.reset()
```

**Total time**: ~11 seconds (instead of ~25s if we tried DeepGram for each)

### Scenario 2: User Records New Clip While Queue is Processing

**Timeline**:
```
Queue processing in background:
├─ Clip 1 (retry) → Whisper → processing...
├─ Clip 2 (retry) → Whisper → queued...
└─ Clip 3 (retry) → Whisper → queued...

User records NEW clip (foreground):
└─ Clip 4 (new):
    ├─ shouldTryDeepgram(isQueueRetry=false) → YES (fresh connection)
    ├─ Try DeepGram → SUCCESS ✅ (VPN off or network changed)
    └─ Time: ~2 seconds

Both processes run in PARALLEL ✅
```

**Why parallel works**:
- Server handles multiple concurrent requests
- Queue retries are separate API calls
- New recording is separate API call
- No blocking

### Scenario 3: VPN Turned Off Mid-Queue

**Timeline**:
```
Queue processing:

Clip 1 (VPN ON):
  ├─ DeepGram → FAIL
  ├─ Circuit OPENS
  └─ Whisper → SUCCESS

User turns VPN OFF (network change)

Clip 2 (VPN OFF):
  ├─ Circuit is OPEN → Skip DeepGram
  └─ Whisper → SUCCESS

Clip 3 (VPN OFF):
  ├─ Circuit is OPEN → Skip DeepGram
  └─ Whisper → SUCCESS

Queue complete → Reset circuit

Next recording (VPN OFF):
  ├─ Circuit CLOSED
  └─ DeepGram → SUCCESS ✅
```

**Circuit resets after queue**, so next recording tries DeepGram again.
