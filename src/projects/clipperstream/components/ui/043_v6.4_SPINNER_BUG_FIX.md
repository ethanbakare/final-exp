# 043_v9 CRITICAL FIX: Spinner Not Spinning During Auto-Retry

**Date**: January 9, 2026  
**Status**: 🔴 **CRITICAL BUG FIXED**  
**Issue**: Spinner shows as static during transcription (not spinning)  

---

## 🐛 Bug Report

### User's Observation
1. ✅ Offline recording shows "Waiting to transcribe" correctly
2. ✅ Coming online changes status to "Transcribing..."
3. ❌ **BUT spinner icon is NOT spinning** (stays static at 40% opacity)
4. ❌ Transcription happens but UI doesn't indicate active HTTP request

### Impact
- **Visual feedback broken**: User can't tell if transcription is actively happening or stuck
- **Affects both screens**:
  - ClipHomeScreen (home screen list)
  - ClipRecordScreen (pending clips view)
- **UX confusion**: Static spinner looks like "waiting" not "actively transcribing"

## 🔍 Root Cause Analysis

### The Spinner Logic

**ClipListItem (cliplist.tsx line 389-392)**:
```typescript
<div className={`status-icon-wrapper ${isActiveRequest ? 'spinning-wrapper' : ''}`}>
  <svg className={`pending-icon ${isActiveRequest ? 'spinning-icon' : ''}`}
```

**ClipOffline (ClipOffline.tsx line 115)**:

**Both require**:
- `status === 'transcribing'` ✅ (was being set correctly)
- `isActiveRequest === true` ❌ (was NEVER set!)

**ClipMasterScreen.tsx Line 189-192** (selectedPendingClips):
```typescript
return children.map(child => ({

  title: child.pendingClipTitle || 'Pending',
  status: (child.status === 'transcribing' ? 'transcribing' : 'waiting'),
  isActiveRequest: activeHttpClipId === child.id  // ← Compares activeHttpClipId

```

**ClipHomeScreen.tsx** (getDisplayClip):
```typescript
const derivedIsActiveRequest =
  activeTranscriptionParentId !== null &&
  clip.id === activeTranscriptionParentId &&
  children.some(c => c.id === activeHttpClipId);  // ← Compares activeHttpClipId
```

**Requires**: `activeHttpClipId` must be set to the child clip ID during HTTP request

```typescript
// ❌ BEFORE (BROKEN):
updateClip(firstClip.id, { status: 'transcribing' });  // Sets status
```

**Auto-retry** (coming online):

- ❌ `isActiveRequest` stays `false`
- ❌ Spinner stays static (40% opacity)

## ✅ The Fix

### File: `ClipMasterScreen.tsx`

**Lines 533-558** (processAllPendingClips):

**BEFORE**:
```typescript
// Clear retrieval attempts on success
audioRetrievalAttempts.delete(firstClip.id);

// Set status to 'transcribing'
updateClip(firstClip.id, { status: 'transcribing' });

// ONE LOOP: Use shared retry logic
const result = await attemptTranscription(audioBlob, {
  maxRapidAttempts: 3,
  useIntervals: true,
  onProgress: (attempt, total) => {
    console.log(`[ProcessPending] Attempt ${attempt}/${total}`);
  }
});
```

**AFTER** (FIXED):
```typescript
// Clear retrieval attempts on success
audioRetrievalAttempts.delete(firstClip.id);

// Set status to 'transcribing'
updateClip(firstClip.id, { status: 'transcribing' });

// ✅ CRITICAL FIX: Track active HTTP request for spinner animation
setActiveTranscriptionParentId(currentParent.id);
setActiveHttpClipId(firstClip.id);

// ONE LOOP: Use shared retry logic
const result = await attemptTranscription(audioBlob, {
  maxRapidAttempts: 3,
  useIntervals: true,
  onProgress: (attempt, total) => {
    console.log(`[ProcessPending] Attempt ${attempt}/${total}`);
  }
});

// ✅ CRITICAL FIX: Clear tracking after transcription completes
setActiveHttpClipId(null);
setActiveTranscriptionParentId(null);
```

### What Changed
1. ✅ **Before HTTP**: Set `activeHttpClipId` to child ID
2. ✅ **Before HTTP**: Set `activeTranscriptionParentId` to parent ID
3. ✅ **After HTTP**: Clear both tracking variables

## 📝 Why This Bug Existed

### History
1. **Manual retry implemented first** (v2.5.4):
   - Included `setActiveHttpClipId` tracking
   - Spinner worked correctly
   
2. **Auto-retry implemented later** (v1.43):
   - Copied status update logic
   - **Forgot to copy tracking logic**
   - Spinner didn't work

### Root Cause
- **Pattern inconsistency**: Two different functions doing similar things but with different implementations
- **Missing code review**: The missing tracking wasn't caught during implementation
- **No automated tests**: No test to verify spinner spins during transcription

## 🎓 Lessons Learned

### For Future Implementation
1. **Shared patterns should be shared code**: If two functions need the same tracking, extract to utility
2. **Visual state requires explicit tracking**: UI indicators (like spinners) need explicit state variables
3. **Test visual feedback**: Verify not just "does it work" but "does user see feedback"

### Pattern to Follow
**Whenever starting HTTP for transcription**:
```typescript
// ✅ ALWAYS DO THIS:
setActiveTranscriptionParentId(parentId);
setActiveHttpClipId(clipId);
try {
  await someTranscriptionFunction();
} finally {
  setActiveHttpClipId(null);
  setActiveTranscriptionParentId(null);
}
```
