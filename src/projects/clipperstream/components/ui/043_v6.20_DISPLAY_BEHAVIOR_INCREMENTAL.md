## Key Insight: Dynamic Discovery

### The Loop is NOT Static!

**WRONG assumption**:
```
"The loop knows there are 5 clips at the start and processes exactly 5"
```

**CORRECT reality**:
```
The loop keeps calling getFirstPendingClipInParent(currentParent) until it returns null
```

**Code (Line 502)**:
```typescript
const firstClip = getFirstPendingClipInParent(currentParent);

if (!firstClip) {
  // No more pending clips
  parentQueue.shift();  // Move to next parent
  continue;
}
```

**What `getFirstPendingClipInParent` does** (Line 447-456):
```typescript
function getFirstPendingClipInParent(parent: Clip): Clip | null {
  const { clips: allClips } = useClipStore.getState();  // ← FRESH data every time!
  const children = allClips.filter(c =>
    c.parentId === parent.id && c.status === 'pending-child'
  );
  
  children.sort((a, b) => a.createdAt - b.createdAt);  // Sort by time
  return children[0] || null;  // Return oldest
}
```

**This function queries Zustand EVERY TIME** - it's not using a cached list!

**A**: ❌ NO - Text appears **incrementally** as each clip finishes

### Q: "Does it know the number of clips at the start?"
**A**: ❌ NO - It discovers clips **dynamically** by querying Zustand each loop iteration

### Q: "Will the new clip be in the right position?"

## Why This Works Perfectly

### 1. Dynamic Discovery
- Loop doesn't cache the pending clip list
- Queries Zustand fresh every iteration
- Picks up newly created clips automatically

### 2. Sequential Processing
- `getFirstPendingClipInParent()` sorts by `createdAt`
- Always returns oldest pending clip first
- Your new clip has newest `createdAt` → Processed last

### 3. Immediate Updates
- `updateClip(currentParent.id, updatedContent)` updates Zustand immediately
- React re-renders the UI automatically
- User sees text grow incrementally

### 4. No Batching
- No "wait until all clips finish" logic
- Each clip updates parent immediately after processing
- Gives user live feedback on progress

## Architectural Pattern

This is **"Stream Processing"** pattern:
- Process items one at a time
- Update results incrementally
- Discover new items dynamically
- No fixed batch size

**Similar to**:
- Real-time log streaming
- Incremental search results
- Progressive loading

**Benefits**:
- User sees progress immediately
- New items can be added mid-stream
- No need to restart if queue changes
- Better UX (feels responsive)
