## 🔍 Root Cause Analysis

### What "fetch failed" Means

The error `Deepgram API error: fetch failed` with **no error code/status** indicates:

1. ❌ **NOT a 400 error** (bad audio format) - would have status code
2. ❌ **NOT a 401 error** (API key) - would have status code
3. ❌ **NOT a codec rejection** - would have specific error message
4. ✅ **IS a network-level fetch failure** - fetch API couldn't complete the request

### Possible Causes

#### 1. **Deepgram SDK Client Reuse Issue** 🎯 **MOST LIKELY**

**Evidence**:
- Online recording: Works (first call to Deepgram)
- Offline recording: Fails (subsequent call to Deepgram)
- Error has no details (suggests SDK client state corruption)

**Theory**: The Deepgram client instance might be getting corrupted or not properly handling multiple requests.

**Code location**: `src/projects/clipperstream/api/deepgramProvider.ts`

---

#### 2. **Request Body/Stream Issue**

**Evidence**:
- Both files are valid WebM
- Server receives correct data
- But something in how we send it to Deepgram fails

**Theory**: The way we're reading and sending the file to Deepgram might not work reliably.

---

#### 3. **Async Race Condition**

**Evidence**:
- Offline recordings trigger auto-retry immediately when going online
- Multiple operations happening simultaneously
- Timing-sensitive failure

**Theory**: Some async operation isn't completing before Deepgram call.

---

#### 4. **Deepgram API Rate Limiting (unlikely)**

**Evidence against this**:
- Online recording works
- Offline recording is the ONLY call
- No 429 status code

## **SOLUTION 1: Fix Deepgram SDK Usage** ⭐ **RECOMMENDED**

### The Problem

Current implementation likely reuses Deepgram client instance incorrectly.

### The Fix

**Create a fresh Deepgram client for each transcription request.**

### Implementation

**File**: `src/projects/clipperstream/api/deepgramProvider.ts`

**Current approach** (likely problematic):
```typescript
// Creating client once and reusing
const deepgram = createClient(apiKey);
const { result, error } = await deepgram.listen.prerecorded.transcribeFile(...)
```

**Fixed approach**:
```typescript
export async function transcribeAudio(
  audioBuffer: Buffer,
  mimeType: string
): Promise<TranscriptionResult> {
  const apiKey = process.env.DEEPGRAM_API_KEY;

  if (!apiKey) {
    throw new Error('Deepgram API key not configured');
  }

  // Create a FRESH client for EACH request
  const deepgram = createClient(apiKey);

  try {
    // Use ReadableStream approach instead of Buffer
    const audioStream = new ReadableStream({
      start(controller) {
        controller.enqueue(audioBuffer);
        controller.close();
      }
    });

    const { result, error } = await deepgram.listen.prerecorded.transcribeFile(
      audioStream,
      {
        model: 'nova-2',
        smart_format: true,
        punctuate: true,
        paragraphs: false,
        utterances: false,
      }
    );

    if (error) {
      throw new Error(`Deepgram error: ${error.message || 'Unknown error'}`);
    }

    // ... rest of processing
  } catch (err) {
    // ... error handling
  }
}
```

### Pros
- ✅ Minimal code change
- ✅ Uses official Deepgram SDK
- ✅ Fixes potential client state issues
- ✅ Industry standard approach

### Cons
- ⚠️ Slightly more overhead (new client per request)
- ⚠️ May not fix issue if problem is elsewhere

## **SOLUTION 2: Direct REST API Call (Bypass SDK)**

### The Problem

Deepgram SDK might have bugs or state management issues.

### The Fix

**Make direct HTTP POST to Deepgram REST API using native fetch.**

### Implementation

**File**: `src/projects/clipperstream/api/deepgramProvider.ts`

**Replace entire file with**:
```typescript
export interface TranscriptionResult {
  transcript: string;
  confidence: number;
  words: Array<{ word: string; start: number; end: number }>;
  duration: number;
}

export async function transcribeAudio(
  audioBuffer: Buffer,
  mimeType: string
): Promise<TranscriptionResult> {
  const apiKey = process.env.DEEPGRAM_API_KEY;

  if (!apiKey) {
    throw new Error('Deepgram API key not configured');
  }

  try {
    // Direct REST API call
    const response = await fetch('https://api.deepgram.com/v1/listen?model=nova-2&smart_format=true&punctuate=true', {
      method: 'POST',
      headers: {
        'Authorization': `Token ${apiKey}`,
        'Content-Type': mimeType,
      },
      body: audioBuffer,
    });

    if (!response.ok) {
      const errorText = await response.text();
      throw new Error(`Deepgram API error (${response.status}): ${errorText}`);
    }

    const result = await response.json();

    // Extract transcript
    const channel = result.results?.channels?.[0];
    const alternative = channel?.alternatives?.[0];

    if (!alternative?.transcript) {
      throw new Error('No transcript in Deepgram response');
    }

    return {
      transcript: alternative.transcript,
      confidence: alternative.confidence || 0,
      words: alternative.words || [],
      duration: result.metadata?.duration || 0,
    };

  } catch (err) {
    console.error('=== DEEPGRAM REST API ERROR ===');
    console.error('Error:', err);
    console.error('================================');

    const errorMessage = err instanceof Error ? err.message : 'Unknown error';
    throw new Error(`Transcription failed: ${errorMessage}`);
  }
}
```

### Pros
- ✅ Complete control over HTTP request
- ✅ No SDK bugs or state issues
- ✅ Easier to debug (can see exact request/response)
- ✅ More reliable (native fetch is battle-tested)
- ✅ Better error messages

### Cons
- ⚠️ Need to handle API changes manually
- ⚠️ Lose some Deepgram SDK features

### Time to Implement
- ~15 minutes

```typescript
import OpenAI from 'openai';
import fs from 'fs';

export interface TranscriptionResult {
  transcript: string;
  confidence: number;
  words: Array<{ word: string; start: number; end: number }>;
  duration: number;
}

export async function transcribeAudio(
  filePath: string,
  mimeType: string
): Promise<TranscriptionResult> {
  const apiKey = process.env.OPENAI_API_KEY;

  if (!apiKey) {
    throw new Error('OpenAI API key not configured');
  }

  const openai = new OpenAI({ apiKey });

  try {
    // OpenAI Whisper expects a file stream
    const audioFile = fs.createReadStream(filePath);

    const transcription = await openai.audio.transcriptions.create({
      file: audioFile,
      model: 'whisper-1',
      response_format: 'verbose_json',
      timestamp_granularities: ['word']
    });

    return {
      transcript: transcription.text,
      confidence: 1.0, // Whisper doesn't provide confidence
      words: transcription.words || [],
      duration: transcription.duration || 0,
    };

  } catch (err) {
    console.error('=== WHISPER API ERROR ===');
    console.error('Error:', err);
    console.error('=========================');

    const errorMessage = err instanceof Error ? err.message : 'Unknown error';
    throw new Error(`Transcription failed: ${errorMessage}`);
  }
}
```
