## 📋 HOW AUTO-RETRY PROCESSES CLIPS

### Core Function: `processAllPendingClips`
**File:** [ClipMasterScreen.tsx:459-682](ClipMasterScreen.tsx#L459-L682)

**Key behavior:**
1. **Processes ONE clip at a time** (line 502: `getFirstPendingClipInParent`)
2. **Sets ONLY that clip to 'transcribing'** (line 562)
3. **Loops through clips sequentially** - not in parallel

**Helper function: `getFirstPendingClipInParent` (line 448-456):**
```typescript
function getFirstPendingClipInParent(parent: Clip): Clip | null {
  const children = allClips.filter(c =>
    c.parentId === parent.id && c.status === 'pending-child'  // ← ONLY 'pending-child'
  );

  children.sort((a, b) => a.createdAt - b.createdAt);
  return children[0] || null;  // ← Returns oldest pending child
}
```

**Critical:** Once a clip's status changes from 'pending-child' to anything else ('transcribing', 'pending-retry', 'audio-corrupted', etc.), this function will skip it and return the next 'pending-child'.

## 🔴 SCENARIO 1: VPN BLOCKED

### What Actually Happens (Code Flow)

**Clip Setup:**
- Parent: "Recording 01"
  - Child A (Clip 001): `status: 'pending-child'`
  - Child B (Clip 002): `status: 'pending-child'`
  - Child C (Clip 003): `status: 'pending-child'`
  - Child D (Clip 004): `status: 'pending-child'`

**Processing Timeline:**

**Step 1:** `processAllPendingClips` starts
```typescript
firstClip = getFirstPendingClipInParent(parent)  // Returns Child A
updateClip(Child A, { status: 'transcribing' })  // Line 562
```

**State:**
- Child A: `status: 'transcribing'`
- Child B, C, D: `status: 'pending-child'` (unchanged)

---

**Step 2:** VPN blocks transcription attempt
```typescript
result = await attemptTranscription(...)  // Fails with error: 'dns-block'

if (result.error === 'dns-block') {  // Line 650
  updateClip(Child A, {
    status: 'pending-retry',   // ← Child A changes from 'transcribing' to 'pending-retry'
    lastError: 'dns-block'     // ← VPN error flag
  });
  await new Promise(resolve => setTimeout(resolve, 30000));  // Wait 30s
  continue;  // Don't rotate parent
}
```

**State:**
- Child A: `status: 'pending-retry', lastError: 'dns-block'`
- Child B, C, D: `status: 'pending-child'` (still unchanged)

---

**Step 3:** Loop continues after 30s wait
```typescript
firstClip = getFirstPendingClipInParent(parent)  // Now returns Child B (Child A is no longer 'pending-child')
updateClip(Child B, { status: 'transcribing' })
```

**State:**
- Child A: `status: 'pending-retry', lastError: 'dns-block'` (VPN blocked)
- Child B: `status: 'transcribing'`
- Child C, D: `status: 'pending-child'`

---

**Step 4:** If VPN still on, Child B also fails
```typescript
updateClip(Child B, {
  status: 'pending-retry',
  lastError: 'dns-block'
});
await new Promise(resolve => setTimeout(resolve, 30000));
```

**State:**
- Child A: `status: 'pending-retry', lastError: 'dns-block'` (VPN blocked)
- Child B: `status: 'pending-retry', lastError: 'dns-block'` (VPN blocked)
- Child C: `status: 'transcribing'`
- Child D: `status: 'pending-child'`

---

**And so on...**

### ❌ NOT What Happens

All clips do NOT show 'VPN blocked' at the same time.

### ✅ What ACTUALLY Happens

**Clips transition ONE AT A TIME:**
1. Child A tries → becomes 'vpn-blocked'
2. Child B tries → becomes 'vpn-blocked'
3. Child C tries → becomes 'vpn-blocked'
4. Child D tries → becomes 'vpn-blocked'

**30 seconds between each attempt** (line 656: `setTimeout(resolve, 30000)`)

## 📊 SUMMARY

### Question 1: VPN Blocked - Do ALL clips show 'VPN blocked'?

**Answer:** ❌ **NO**

**Reality:**
- Clips become 'vpn-blocked' **ONE AT A TIME** as they attempt transcription
- 30-second delay between each attempt
- After 2 minutes (4 clips × 30s), you might see:
  - 2-3 clips showing 'VPN blocked' (already tried)
  - 1 clip showing 'Transcribing...' (currently trying)
  - Remaining clips showing 'Waiting' (haven't tried yet)

**Parent:**
- Shows 'VPN Blocked' if ANY child has 'dns-block' error AND no child is currently transcribing
- Shows 'Transcribing...' if ANY child is currently `status: 'transcribing'` (higher priority)

---

### Question 2: Transcribing - Only first clip shows spinner?

**Answer:** ✅ **YES** (for children in record screen)

**Reality:**
- **Record Screen:** Only ONE child shows spinning icon at a time (the one with `status: 'transcribing'`)
- **Home Screen:** Parent ALWAYS shows spinning icon when ANY child is `status: 'transcribing'`

**Sequential processing:**
1. Child A transcribes (spinner spins) → succeeds → deleted
2. Child B transcribes (spinner spins) → succeeds → deleted
3. Child C transcribes (spinner spins) → succeeds → deleted
4. All done - parent now has complete content

---

## 🎯 KEY TAKEAWAYS

1. **ONE clip processes at a time** - never parallel
2. **VPN blocking is GRADUAL** - clips fail one by one over time
3. **Parent inherits highest-priority child status** - transcribing > vpn-blocked > retry-pending > pending
4. **Spinner control is SIMPLE** - `spinning={status === 'transcribing'}` (no `isActiveRequest` dependency)
5. **Parent ALWAYS shows spinner** when any child is transcribing (auto-retry doesn't set `activeHttpClipId`)
