## What Was Discovered

### User's Investigation Process

1. **Initial report**: Nav bar doesn't update after pending clip transcribes
2. **Second test**: Discovered state DOES update (buttons appear after navigation)
3. **Key insight**: Problem is real-time reactivity, not missing state
4. **Third test**: Asked about live recording architecture
5. **Breakthrough**: Traced exact flow of how live recording updates nav bar
6. **Root cause confirmed**: Pending clips use different code path that doesn't update nav bar

### Architectural Understanding Achieved

**Live Recording Flow** (Lines 907, 910, 1213):
```
Record → Create clip → Format → Update nav bar UNCONDITIONALLY
                                 ↓
                         setRecordNavState('complete')
```

**Pending Clips Flow** (Lines 592, 611):
```
Auto-retry → Transcribe → Format → Update content → ❌ NO NAV BAR UPDATE
```

**The Gap**: Pending clips update content but never call `setRecordNavState('complete')`

## Dependency Verification

### What's Already Correct

✅ **Line 472** uses industry best practice:
```typescript
const { clips: allClips, updateClip, deleteClip, getClipById } = useClipStore.getState();
```
These functions are fetched fresh each time, don't need to be in deps.

✅ **Current dependencies** are correct:
- `formatChildTranscription` - useCallback that needs to be in deps
- `setIsProcessingPending` - Zustand setter
- `setAudioRetrievalAttempt` - Zustand setter
- `deleteAudioRetrievalAttempt` - Zustand setter

### What We're Adding

✅ **`currentClipId`** (Line 66):
- Source: `useState<string | null>(null)`
- Type: React state **VALUE**
- Why needed: Callback needs fresh value to compare against `currentParent.id`
- Without it: Stale closure → condition always uses old value → nav bar never updates

✅ **`setRecordNavState`** (Line 158):
- Source: `useState<RecordNavState>('record')`
- Type: React state **SETTER**
- Why needed: Callback calls it to update nav bar
- Note: React guarantees setState functions are stable, but include for exhaustive-deps lint rule
