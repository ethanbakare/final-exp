## 🐛 REGRESSION FROM v2.5

### What Happened

**v2.4 architecture change** separated parents from children:
- **Parents**: Containers with `status: null`, NO audioId, shown on home screen
- **Children**: Actual clips with `status: 'pending-child'`, audioId, hidden from home screen

**Unintended consequence:**
- Home screen now shows parents (which have `status: null`)
- "Waiting to transcribe" indicator disappeared (parents don't have pending status)
- User has no visual feedback that clips are waiting to transcribe

## ✅ THE FIXES

### Fix #1: Add Derived Status to ClipHomeScreen

**File:** [ClipHomeScreen.tsx](ClipHomeScreen.tsx)

**Current code (approximate location - find where clips are mapped to ClipListItem):**

```typescript
{clips.map((clip) => (
  <ClipListItem
    key={clip.id}
    clip={clip}           // ← clip.status is null for parents
    onClick={() => onClipClick(clip.id)}
    onDelete={() => handleDeleteClip(clip.id)}
  />
))}
```

**AFTER (v2.5.1 - Add derived status):**

```typescript
{clips.map((clip) => {
  // v2.5.1 FIX: Derive status from children for parent clips
  // Parents have status: null, but should show "pending" if children are pending
  const allClips = getClips();
  const children = allClips.filter(c => c.parentId === clip.id);
  const hasPendingChildren = children.some(c =>
    c.status === 'pending-child' || c.status === 'transcribing'
  );

  // Create clip with derived status for display
  const displayClip = hasPendingChildren
    ? { ...clip, status: 'pending' as const }
    : clip;

  return (
    <ClipListItem
      key={clip.id}
      clip={displayClip}    // ← Now has derived status
      onClick={() => onClipClick(clip.id)}
      onDelete={() => handleDeleteClip(clip.id)}
    />
  );
})}
```

**Alternative (cleaner):**

If you want to avoid inline logic, add a helper function at the top of ClipHomeScreen:

```typescript
// v2.5.1 FIX: Helper to derive status from children
const getDisplayClip = useCallback((clip: Clip, allClips: Clip[]): Clip => {
  // If clip has children with pending status, derive 'pending' status
  const children = allClips.filter(c => c.parentId === clip.id);
  const hasPendingChildren = children.some(c =>
    c.status === 'pending-child' || c.status === 'transcribing'
  );

  if (hasPendingChildren) {
    return { ...clip, status: 'pending' as const };
  }

  return clip;
}, []);

// Then in render:
{clips.map((clip) => {
  const displayClip = getDisplayClip(clip, getClips());

  return (
    <ClipListItem
      key={clip.id}
      clip={displayClip}
      onClick={() => onClipClick(clip.id)}
      onDelete={() => handleDeleteClip(clip.id)}
    />
  );
})}
```

  });

  if (pendingClips.length === 0) {
    log.debug('No pending clips to retry');
    return;
  }

  log.info('Found pending clips for auto-retry', { count: pendingClips.length });

  for (const clip of pendingClips) {
    // ... (rest of retry logic - lines 526-584)
  }
}, [isRecording, refreshClips, transcribeRecording]);

- Now checks for `status === 'pending-child'` (children with audioId)
