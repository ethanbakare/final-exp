# 043_v6.8 OPACITY BUG FIX

**Date**: January 9, 2026
**Status**: ✅ **FIXED**
**Issue**: Spinner spinning but showing muted opacity (40%) instead of full white

---

## 🎯 THE BUG

After implementing Option A (spinner spins based on `status` alone), the spinner was spinning correctly but showing **40% opacity** instead of **full white** when `status === 'transcribing'`.

---

## 🔍 ROOT CAUSE

**Incomplete Fix**: When removing `isActiveRequest` dependency from line 115, we forgot to also remove it from line 114.

### The Problematic Code (BEFORE)

**ClipOffline.tsx line 114-115**:
```typescript
// Line 114: OLD logic still checking isActiveRequest ❌
<div className={`... ${status === 'waiting' || status === 'retry-pending' || (status === 'transcribing' && isActiveRequest === false) ? 'waiting-opacity' : ''}`}>

  // Line 115: NEW logic (just changed) ✅
  <TranscribeBig spinning={status === 'transcribing'} />
</div>
```

**The Problem**:
- Line 115 says: "Spin when `status === 'transcribing'`" ✅
- Line 114 says: "Apply muted opacity when `status === 'transcribing' && isActiveRequest === false`" ❌
- Since `isActiveRequest` was not being set, it was always `undefined` (falsy)
- So: `status === 'transcribing' && false` → applies `waiting-opacity` class (40%)
- Result: Spinner spinning but muted ❌

---

## ✅ THE FIX

**ClipOffline.tsx line 114**:
```typescript
// BEFORE:
${status === 'waiting' || status === 'retry-pending' || (status === 'transcribing' && isActiveRequest === false) ? 'waiting-opacity' : ''}

// AFTER:
${status === 'waiting' || status === 'retry-pending' ? 'waiting-opacity' : ''}
```

**Removed**: `|| (status === 'transcribing' && isActiveRequest === false)`

**Now**:
- `status === 'waiting'` → muted opacity (40%) ✅
- `status === 'retry-pending'` → muted opacity (40%) ✅
- `status === 'transcribing'` → **full white opacity (100%)** ✅

## 🐛 ADDITIONAL FIX: TypeScript Type Errors

### Issue #1: Invalid `'failed'` Status

**Error**:
```
Type '"transcribing" | "waiting" | "failed"' is not assignable to type 'ClipOfflineStatus'
Type '"failed"' is not assignable to type 'ClipOfflineStatus'
```

**Root Cause**: PendingClip interface included `'failed'` which is not a valid ClipOfflineStatus.

**Fix**: [ClipRecordScreen.tsx:28](final-exp/src/projects/clipperstream/components/ui/ClipRecordScreen.tsx#L28)
```typescript
// BEFORE:
status?: 'waiting' | 'transcribing' | 'failed';

// AFTER:
status?: 'waiting' | 'retry-pending' | 'transcribing' | 'vpn-blocked' | 'audio-corrupted' | 'no-audio-detected';
```

### Issue #2: Incomplete Status Mapping

**Root Cause**: `selectedPendingClips` was only mapping `'transcribing'` vs everything else as `'waiting'`, not handling all the possible clip statuses.

**Fix**: [ClipMasterScreen.tsx:186-210](final-exp/src/projects/clipperstream/components/ui/ClipMasterScreen.tsx#L186-L210)
```typescript
// BEFORE (simplified):
status: (child.status === 'transcribing' ? 'transcribing' : 'waiting')

// AFTER (complete mapping):
let mappedStatus = 'waiting';

if (child.status === 'transcribing') {
  mappedStatus = 'transcribing';
} else if (child.status === 'pending-retry') {
  mappedStatus = child.lastError === 'dns-block' ? 'vpn-blocked' : 'retry-pending';
} else if (child.status === 'audio-corrupted') {
  mappedStatus = 'audio-corrupted';
} else if (child.status === 'no-audio-detected') {
  mappedStatus = 'no-audio-detected';
} else if (child.status === 'pending-child') {
  mappedStatus = 'waiting';
}
```

**Now Handles**:
- ✅ `'pending-child'` → `'waiting'`
- ✅ `'transcribing'` → `'transcribing'`
- ✅ `'pending-retry'` → `'retry-pending'` (or `'vpn-blocked'` if dns-block)
- ✅ `'audio-corrupted'` → `'audio-corrupted'`
- ✅ `'no-audio-detected'` → `'no-audio-detected'`
