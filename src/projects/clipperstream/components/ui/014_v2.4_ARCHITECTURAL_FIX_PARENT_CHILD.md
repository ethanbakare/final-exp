## 🔍 ROOT CAUSE ANALYSIS (After Reading Full Code)

### The Fundamental Architectural Flaw

After reading [ClipMasterScreen.tsx](ClipMasterScreen.tsx), I've identified the root cause of ALL three bugs:

**The parent recording IS Clip 001.**

#### Current (BROKEN) Architecture:

When you record offline for the first time:

```typescript
// Line 1360-1379 in ClipMasterScreen.tsx
const nextNumber = getNextRecordingNumber(getClips());  // "Recording 01"
const newClip = createNewClip('', nextNumber, '');      // Creates clip
clipIdToUpdate = newClip.id;
setCurrentClipId(newClip.id);

// UPDATE PARENT WITH FIRST PENDING CLIP
updateClipById(clipIdToUpdate, {
  audioId: audioId,                    // ← PARENT HAS AUDIO!
  duration: formatDuration(duration),
  status: 'pending',
  pendingClipTitle: 'Clip 001',        // ← PARENT IS CLIP 001!
  parentId: undefined
});
```

**Result in storage:**
```javascript
{
  id: "clip-1766876478739-rlnx5i6te",
  title: "Recording 01",          // Parent title
  pendingClipTitle: "Clip 001",   // Also Clip 001
  audioId: "audio-...",           // Has audio!
  status: 'pending',              // Pending status
  parentId: undefined             // Is parent
}
```

**When Clip 001 transcribes:**
```javascript
{
  id: "clip-1766876478739-rlnx5i6te",
  title: "Recording 01",
  pendingClipTitle: "Clip 001",        // Still marked as pending clip
  formattedText: "Clip 001 text...",   // Now has text!
  status: null,                        // Transcribed
  parentId: undefined
}
```

This dual identity (parent + Clip 001) causes ALL three bugs.

## 🐛 How This Causes Each Bug

### Bug #1: Children Get Wrong Title ("Recording 02" instead of "Recording 01")

**Location:** [ClipMasterScreen.tsx:1409-1410](ClipMasterScreen.tsx#L1409-L1410)

```typescript
// Creating CHILD (lines 1391-1437)
const nextNumber = getNextRecordingNumber(getClips());  // ← PROBLEM!
const childClip = createNewClip('', nextNumber, '');    // ← Creates "Recording 02"

updateClipById(clipIdToUpdate, {
  audioId: audioId,
  duration: formatDuration(duration),
  status: 'pending-child',
  pendingClipTitle: nextPendingTitle,  // "Clip 002"
  parentId: parentClip.id
});
```

**Why it happens:**
- `createNewClip('', nextNumber, '')` ALWAYS creates a new clip with a new recording number
- `nextNumber = getNextRecordingNumber(getClips())` returns "Recording 02"
- Child gets `title: "Recording 02"` even though parent is "Recording 01"

**Why v2.3.3 failed:**
- My fix tried to inherit parent title AFTER `createNewClip` already set it to "Recording 02"
- Changing the title after creation doesn't work because `createNewClip` bakes it in

---

### Bug #2: Transcript State Leak (Recording 03 appears in Recording 01)

**Root Cause:** Parent IS Clip 001, so when Clip 001 transcribes:

```javascript
// Parent = Clip 001 transcribes
{
  id: "parent-id",
  title: "Recording 01",
  pendingClipTitle: "Clip 001",        // ← Still has pending marker
  formattedText: "Clip 001 text...",   // ← Now has text
  status: null
}
```

When you navigate to this clip in `handleClipClick` ([ClipMasterScreen.tsx:218-235](ClipMasterScreen.tsx#L218-L235)):

```typescript
if (clip.content) {
  // Transcribed clip - show in complete state
  setSelectedClip(clip);  // ← Sets parent as selectedClip
  setSelectedPendingClips([]);
  setContentBlocks([{
    id: clip.id,
    text: clip.content,    // ← Shows parent's text
    animate: false
  }]);
}
```

**The leak happens because:**
1. Parent has `formattedText` (from Clip 001 transcription)
2. When you record Clip 002 in a different recording (Recording 03), background transcription completes
3. The transcription handler updates the wrong clip because parent still has `pendingClipTitle`

**Why v2.3.3 failed:**
- My fix tried to prevent updating `currentClipId` during background transcription
- But the real problem is that parent shouldn't have `formattedText` at all
- Parent should be container-only, not an actual clip with audio/text

## ✅ THE ARCHITECTURAL FIX (v2.4)

### Proposed Architecture

**Separate parent (container) from Clip 001 (first child):**

```
Recording 01 (PARENT - container ONLY)
  ├─ title: "Recording 01"
  ├─ status: null (not 'pending')
  ├─ NO audioId
  ├─ NO pendingClipTitle
  ├─ NO content/formattedText/rawText
  └─ Children:
      ├─ Clip 001 (CHILD - has audio)
      │   ├─ id: "clip-..."
      │   ├─ title: "Recording 01" (inherited from parent)
      │   ├─ parentId: parent.id
      │   ├─ pendingClipTitle: "Clip 001"
      │   ├─ audioId: "audio-..."
      │   └─ status: 'pending-child'
      ├─ Clip 002 (CHILD)
      │   ├─ title: "Recording 01" (inherited)
      │   ├─ parentId: parent.id
      │   ├─ pendingClipTitle: "Clip 002"
      │   └─ ...
```

**Key principle:** Parents are containers ONLY. Children have audio.

## 🔧 IMPLEMENTATION

### Change #1: Offline Handler - Create Parent + First Child Separately

**File:** [ClipMasterScreen.tsx:1360-1390](ClipMasterScreen.tsx#L1360-L1390)

**BEFORE (v2.3.3 - Parent IS Clip 001):**

```typescript
if (shouldCreateParent) {
  // Create PARENT recording
  const nextNumber = getNextRecordingNumber(getClips());
  const newClip = createNewClip('', nextNumber, '');
  clipIdToUpdate = newClip.id;
  setCurrentClipId(newClip.id);

  // Update parent with first pending clip
  updateClipById(clipIdToUpdate, {
    audioId: audioId,               // ← PARENT HAS AUDIO
    duration: formatDuration(duration),
    status: 'pending',
    pendingClipTitle: 'Clip 001',   // ← PARENT IS CLIP 001
    parentId: undefined
  });
}
```

**AFTER (v2.4 - Parent = container, Clip 001 = separate child):**

```typescript
if (shouldCreateParent) {
  // Step 1: Create PARENT (container only)
  const nextNumber = getNextRecordingNumber(getClips());
  const parentClip = createNewClip('', nextNumber, '');

  log.info('Created PARENT container for offline recording', {
    parentId: parentClip.id,
    title: nextNumber
  });

  // Parent remains as-is (no audioId, no pendingClipTitle, no status)
  // Default status from createNewClip is null, which is correct

  // Step 2: Create FIRST CHILD (Clip 001)
  const firstChildId = `clip-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;

  // Manually create child clip object (DON'T use createNewClip)
  const firstChild: Clip = {
    id: firstChildId,
    title: parentClip.title,        // Inherit parent's title "Recording 01"
    content: '',
    date: new Date().toLocaleDateString('en-US', {
      month: 'short',
      day: 'numeric',
      year: 'numeric'
    }),
    currentView: 'formatted',
    // Child-specific fields:
    parentId: parentClip.id,        // Link to parent
    status: 'pending-child',
    pendingClipTitle: 'Clip 001',
    audioId: audioId,
    duration: formatDuration(duration)
  };

  // Save child to storage
  const allClips = getClips();
  sessionStorage.setItem('clips', JSON.stringify([firstChild, ...allClips]));
  refreshClips();

  log.info('Created FIRST CHILD for offline recording', {
    childId: firstChild.id,
    parentId: parentClip.id,
    title: firstChild.pendingClipTitle
  });

  // Set currentClipId to PARENT (not child)
  // This allows subsequent recordings to append as children
  setCurrentClipId(parentClip.id);
  clipIdToUpdate = firstChild.id;  // For selectedPendingClips update below

  // Add child to selectedPendingClips
  const pendingClip = clipToPendingClip(firstChild);
  setSelectedPendingClips([pendingClip]);
}
```

**Why this works:**
- Parent is created with just `title: "Recording 01"`, no audio, no pendingClipTitle
- First child is created separately with `parentId: parent.id`, `title: "Recording 01"` (inherited)
- Parent never gets `formattedText` because it has no `audioId`
- Clean separation of concerns

### Change #3: handleClipClick - Load Children for Transcribed Files

**File:** [ClipMasterScreen.tsx:218-235](ClipMasterScreen.tsx#L218-L235)

**BEFORE (v2.3.3 - Doesn't load children for transcribed files):**

```typescript
const handleClipClick = useCallback((clipId: string) => {

    setAnimationVariant('fade');

  } else {
    // Pending clip - load children
    // ... (lines 238-294)
  }
}, [clips]);
```

**AFTER (v2.4 - ALWAYS load children):**

```typescript
const handleClipClick = useCallback((clipId: string) => {
  const allClips = getClips();
  const clip = allClips.find(c => c.id === clipId);
  if (!clip) return;

  // ALWAYS set currentClipId so subsequent recordings append correctly
  setCurrentClipId(clipId);

  // ALWAYS check for children (fixes Bug #3 - pending clips disappear)
  const children = allClips.filter(c => c.parentId === clipId);

  if (children.length > 0) {
    // v2.3.2 FIX: Sort children by creation time
    const sortedChildren = children.sort((a, b) => {
      const timestampA = parseInt(a.id.split('-')[1], 10) || 0;
      const timestampB = parseInt(b.id.split('-')[1], 10) || 0;
      return timestampA - timestampB;
    });

    // Convert to pending clip format
    const pendingClips = sortedChildren.map(child => ({
      id: child.id,
      title: child.pendingClipTitle || 'Pending',
      time: child.duration || '0:00',
      status: child.status === 'transcribing' ? 'transcribing' : 'waiting',
      isActiveRequest: isActiveRequest && currentClipId === child.id
    }));

    setSelectedPendingClips(pendingClips);

    log.info('Loaded parent with children', {
      parentId: clipId,
      childCount: children.length,
      childOrder: pendingClips.map(p => p.title)
    });
  } else {
    setSelectedPendingClips([]);
  }

  if (clip.content) {
    // Transcribed clip - show in complete state
    setSelectedClip(clip);
    setIsAppendMode(false);
    setAppendBaseContent('');
    resetRecording();
    setContentBlocks([{
      id: clip.id,
      text: clip.content,
      animate: false
    }]);
    setAnimationVariant('fade');
    setRecordNavState('complete');
    setActiveScreen('record');
  } else {
    // Pending clip - show in waiting state
    setSelectedClip(null);
    setAnimationVariant('fade');
    setRecordNavState('record');
    setActiveScreen('record');
  }
}, [clips, isActiveRequest, currentClipId, getClips, resetRecording]);
```

**Why this works:**
- Children are loaded for BOTH transcribed and pending files
- When you click "Mary's Lamb" (transcribed), it shows content AND loads pending children
- Fixes Bug #3 completely

### Critical: Don't Use createNewClip() for Children

**Why:**
```typescript

    id: `clip-${Date.now()}-...`,
    title: finalTitle,  // ← Baked into object
    content,
    // ...
  };

```

**Solution:** Manually create child clip object with inherited parent title.
