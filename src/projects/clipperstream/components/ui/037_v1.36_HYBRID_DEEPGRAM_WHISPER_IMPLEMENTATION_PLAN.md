## Overview

Implement intelligent fallback from Deepgram to OpenAI Whisper when VPN/DNS issues are detected, with session-based state management to avoid repeated failures.

---

## Problem Statement

**Current Issue**:
- VPN blocks DNS resolution for `api.deepgram.com`
- All transcriptions fail with `ENOTFOUND` error
- User must manually disable VPN to proceed

**Goal**:
Automatically fallback to OpenAI Whisper when Deepgram DNS fails, making the app work seamlessly with VPN enabled.

---

## User Questions & Answers

### Q1: Does it fail with Deepgram every time before using OpenAI?

**Answer**: NO - Smart session-based fallback

**Strategy**:
```
1st clip: Tries Deepgram → DNS error → Switches to Whisper
2nd-5th clips: Uses Whisper directly (no Deepgram retry)
After idle time: Resets to Deepgram for next session
```

**Why**: Avoid wasting time and API calls on repeated failures.

---

### Q2: If 5 clips in queue, do all switch to OpenAI?

**Answer**: YES - Once DNS error detected, ALL pending clips use Whisper

**Logic**:
```typescript
useWhisperFallback = false;

// First clip fails
Clip 1: Deepgram → DNS Error → useWhisperFallback = true → Whisper ✅

// Remaining clips skip Deepgram
Clip 2: useWhisperFallback = true → Whisper directly ✅
Clip 3: useWhisperFallback = true → Whisper directly ✅
Clip 4: useWhisperFallback = true → Whisper directly ✅
Clip 5: useWhisperFallback = true → Whisper directly ✅
```

**Result**: Only first clip wastes time on Deepgram failure.

---

### Q3: New recording while queue is processing - does it join queue?

**Answer**: NO - New online recordings process in PARALLEL

**Architecture**:
```
Auto-retry Queue (Background):
├─ Clip1 (pending) → Processing...
├─ Clip2 (pending) → Processing...
└─ Clip3 (pending) → Waiting...

User Records New (NOW):
└─ Clip4 → Process immediately (parallel track)
```

**Why**: User shouldn't wait for old pending clips. Good UX.

---

### Q4: Does new online recording use Deepgram or OpenAI?

**Answer**: Checks fallback state

**Logic**:
```typescript
if (useWhisperFallback && timeSinceLastFallback < 5min) {
  // VPN likely still active, use Whisper
  return await transcribeWithWhisper(...);
} else {
  // Normal flow, try Deepgram first
  return await transcribeWithDeepgram(...);
}
```

**Why**: If VPN is still active, Deepgram will fail again. Save time.

---

### Q5: When does it switch back to Deepgram?

**Answer**: Time-based reset (5 minutes)

**Strategy**:
```typescript
// After 5 minutes of inactivity, reset fallback
if (Date.now() - lastFallbackTime > 5 * 60 * 1000) {
  useWhisperFallback = false;  // Try Deepgram again
}
```

**Why**: 
- User might disable VPN during idle time
- Deepgram is faster and cheaper (prefer when available)
- 5 minutes is reasonable "session" duration

## Architecture Design

### State Management

```typescript
// In-memory state (not persisted)
interface FallbackState {
  useWhisperFallback: boolean;     // Current fallback status
  lastFallbackTime: number;        // Timestamp of last DNS error
  consecutiveDeepgramFailures: number;  // Track failure patterns
}

const fallbackState: FallbackState = {
  useWhisperFallback: false,
  lastFallbackTime: 0,
  consecutiveDeepgramFailures: 0,
};

const FALLBACK_RESET_MS = 5 * 60 * 1000; // 5 minutes
```

**Why in-memory (not persisted)**:
- ✅ VPN state changes between app sessions
- ✅ Don't want to "remember" old VPN issues
- ✅ Each app session starts fresh

### Core Transcription Function

```typescript
async function transcribeWithFallback(
  audioBuffer: Buffer,
  mimeType: string,
  deepgramKey: string,
  openaiKey: string
): Promise<TranscriptionResult> {
  
  // Step 1: Check if fallback should be reset
  const timeSinceLastFallback = Date.now() - fallbackState.lastFallbackTime;
  if (timeSinceLastFallback > FALLBACK_RESET_MS) {
    fallbackState.useWhisperFallback = false;
    fallbackState.consecutiveDeepgramFailures = 0;
  }
  
  // Step 2: If fallback is active, use Whisper directly
  if (fallbackState.useWhisperFallback) {
    console.log('🔄 Using Whisper (Deepgram previously failed with DNS error)');
    return await transcribeWithWhisper(audioBuffer, mimeType, openaiKey);
  }
  
  // Step 3: Normal flow - try Deepgram first
  try {
    console.log('🎯 Attempting Deepgram transcription');
    const result = await transcribeWithDeepgram(audioBuffer, mimeType, deepgramKey);
    
    // Reset failure counter on success
    fallbackState.consecutiveDeepgramFailures = 0;
    
    return result;
    
  } catch (err) {
    // Step 4: Check if it's a DNS/VPN error
    const isDNSError = 
      err instanceof Error && (
        err.message.includes('ENOTFOUND') ||
        err.message.includes('ETIMEDOUT') ||
        err.message.includes('DNS lookup failed') ||
        err.message.includes('Cannot reach Deepgram')
      );
    
    if (isDNSError) {
      console.error('⚠️ Deepgram DNS/VPN error detected');
      console.log('📝 Switching to OpenAI Whisper fallback');
      
      // Activate fallback for this session
      fallbackState.useWhisperFallback = true;
      fallbackState.lastFallbackTime = Date.now();
      fallbackState.consecutiveDeepgramFailures++;
      
      // Use Whisper as fallback
      return await transcribeWithWhisper(audioBuffer, mimeType, openaiKey);
    }
    
    // If it's not a DNS error, throw it (e.g., invalid audio, API key issues)
    throw err;
  }
}
```

### Deepgram Transcription (Existing)

```typescript
async function transcribeWithDeepgram(
  audioBuffer: Buffer,
  mimeType: string,
  apiKey: string
): Promise<TranscriptionResult> {
  // This is our existing direct REST API implementation
  // (Already implemented in v1.30)
  
  const controller = new AbortController();
  const timeoutId = setTimeout(() => controller.abort(), 30000);
  
  try {
    const response = await fetch(
      'https://api.deepgram.com/v1/listen?model=nova-3&language=en-US&smart_format=true&punctuate=true',
      {
        method: 'POST',
        headers: {
          'Authorization': `Token ${apiKey}`,
          'Content-Type': mimeType,
        },
        body: audioBuffer,
        signal: controller.signal,
      }
    );
    
    clearTimeout(timeoutId);
    
    if (!response.ok) {
      const errorData = await response.json();
      throw new Error(`Deepgram API error (HTTP ${response.status}): ${JSON.stringify(errorData)}`);
    }
    
    const result = await response.json();
    const transcript = result.results?.channels?.[0]?.alternatives?.[0]?.transcript || '';
    const duration = result.metadata?.duration || 0;
    
    if (!transcript.trim()) {
      throw new Error('No speech detected in recording. Please try again and speak clearly.');
    }
    
    return {
      transcript: transcript.trim(),
      duration,
    };
    
  } catch (err) {
    clearTimeout(timeoutId);
    throw err;
  }
}
```

### OpenAI Whisper Transcription (NEW)

```typescript
import { createReadStream, writeFileSync, unlinkSync } from 'fs';
import { tmpdir } from 'os';
import { join } from 'path';
import { randomBytes } from 'crypto';

async function transcribeWithWhisper(
  audioBuffer: Buffer,
  mimeType: string,
  apiKey: string
): Promise<TranscriptionResult> {
  // Whisper requires file upload (not buffer)
  // Save buffer to temp file
  
  const tempFileName = `whisper-${randomBytes(16).toString('hex')}.webm`;
  const tempFilePath = join(tmpdir(), tempFileName);
  
  try {
    // Write buffer to temp file
    writeFileSync(tempFilePath, audioBuffer);
    
    console.log('🎤 Sending to OpenAI Whisper API');
    
    // Create FormData for file upload
    const FormData = (await import('form-data')).default;
    const formData = new FormData();
    formData.append('file', createReadStream(tempFilePath));
    formData.append('model', 'whisper-1');
    formData.append('language', 'en');
    formData.append('response_format', 'verbose_json'); // Get duration
    
    // Call OpenAI Whisper API
    const response = await fetch('https://api.openai.com/v1/audio/transcriptions', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${apiKey}`,
        ...formData.getHeaders(),
      },
      body: formData,
    });
    
    if (!response.ok) {
      const errorText = await response.text();
      throw new Error(`Whisper API error (HTTP ${response.status}): ${errorText}`);
    }
    
    const result = await response.json();
    
    // Clean up temp file
    unlinkSync(tempFilePath);
    
    // Whisper response format
    const transcript = result.text || '';
    const duration = result.duration || 0;
    
    if (!transcript.trim()) {
      throw new Error('No speech detected in recording. Please try again and speak clearly.');
    }
    
    console.log('✅ Whisper transcription successful');
    
    return {
      transcript: transcript.trim(),
      duration,
    };
    
  } catch (err) {
    // Clean up temp file on error
    try {
      unlinkSync(tempFilePath);
    } catch {}
    
    throw err;
  }
}
```

## Queue Processing Scenario

### Example: 5 Pending Clips (VPN ON)

```
Initial State:
  useWhisperFallback = false
  Queue: [Clip1, Clip2, Clip3, Clip4, Clip5]

─────────────────────────────────────────────

Processing Clip1:
  ↓
  Check fallback: false
  ↓
  Try Deepgram
  ↓
  DNS Error (ENOTFOUND api.deepgram.com)
  ↓
  Set useWhisperFallback = true
  ↓
  Fallback to Whisper
  ↓
  SUCCESS ✅ (took ~8 seconds)

─────────────────────────────────────────────

Processing Clip2:
  ↓
  Check fallback: true
  ↓
  Skip Deepgram, use Whisper directly
  ↓
  SUCCESS ✅ (took ~8 seconds)

─────────────────────────────────────────────

Processing Clip3:
  ↓
  Check fallback: true
  ↓
  Use Whisper directly
  ↓
  SUCCESS ✅ (took ~8 seconds)

─────────────────────────────────────────────

Processing Clip4 & Clip5:
  ↓
  Same as Clip3 (Whisper directly)
  ↓
  SUCCESS ✅

─────────────────────────────────────────────

5 minutes later (idle):
  ↓
  Fallback timer expires
  ↓
  useWhisperFallback = false
  ↓
  Next recording tries Deepgram again
```

**Total time saved**: ~32 seconds (4 clips × 8s each avoided Deepgram timeout)

## Error Handling

### What if BOTH Deepgram AND Whisper fail?

```typescript
try {
  return await transcribeWithDeepgram(...);
} catch (deepgramError) {
  if (isDNSError(deepgramError)) {
    try {
      return await transcribeWithWhisper(...);
    } catch (whisperError) {

      throw new Error(
        'Transcription failed with both services. ' +
        'Please check your internet connection. ' +
```
