# FIX #2: Clip Sort Order (MVP - Creation Time Only)
## Corrects False Claims in Original Patch

---

## What Was Wrong in Original Patch

**Original Claim (INCORRECT)**:
> "The `createdAt` field is updated every time you interact with a clip (add new transcription). So `createdAt` represents the **last interaction time**, not the original creation time."

**Builder Verification (CORRECT)**:

**Append Mode** (lines 530-533 in ClipMasterScreen.tsx):
```typescript
updateClip(currentClipId, {
  rawText: existingClip.rawText + ' ' + rawText,
  status: 'formatting'
  // ❌ NO createdAt update!
});
```

**New Clip Mode** (line 540):
```typescript
const newClip: Clip = {
  createdAt: Date.now(),  // ✅ Only set on creation, NEVER updated
  // ...
};
```

**Builder's Verdict**:
- ✅ `createdAt` is ONLY set when creating new clips
- ✅ `createdAt` is NEVER updated during append operations
- ❌ Therefore, "last touched = top position" does NOT currently work
- ❌ Test 2 from original patch would FAIL

---

## The Bug (Actual)

**File**: `ClipHomeScreen.tsx`
**Location**: Lines 173-176

Clips are not sorted at all, and children (pending clips) are shown alongside parents.

### Current Code
```typescript
const filteredClips = clips.filter(clip =>
  clip.title.toLowerCase().includes(searchQuery.toLowerCase())
);
```

**Problems**:
- No `.filter(clip => !clip.parentId)` - shows both parents and children ❌
- No `.sort()` call - random/undefined order ❌

---

## The Fix (MVP - Correct Expectations)

**Builder's Option B: Sort by Creation Time (MVP)**

This is the **simplest, correct** solution that works NOW without schema changes.

### BEFORE (Wrong)
```typescript
const filteredClips = clips.filter(clip =>
  clip.title.toLowerCase().includes(searchQuery.toLowerCase())
);
```

### AFTER (Correct - MVP)
```typescript
// Filter and sort clips (newest created first, parents only)
const filteredClips = clips
  .filter(clip => !clip.parentId)  // Only parent clips (clip files), not pending clips
  .filter(clip =>
    clip.title.toLowerCase().includes(searchQuery.toLowerCase())
  )
  .sort((a, b) => b.createdAt - a.createdAt);  // Newest created clip first
```

---

## What This Actually Does

✅ **Filters out child clips** (pending clips inside parent)
✅ **Applies search filter**
✅ **Sorts by creation time** (newest clip created = top)
⚠️ **Does NOT give "last interacted" behavior** (that requires schema change)

## Future Enhancement: "Last Touched" Behavior

**Builder's Option A: Add lastModifiedAt Field (Recommended for v2)**

If you want "last touched = top position" behavior, you need:

### Schema Change
```typescript
// In clipStore.ts - Update Clip interface
export interface Clip {
  createdAt: number;        // Never changes (original creation)
  lastModifiedAt?: number;  // Updates on every change (NEW FIELD)
  // ... rest
}
```

### Code Change - New Clips
```typescript
// When creating new clip
const newClip: Clip = {
  createdAt: Date.now(),
  lastModifiedAt: Date.now(),  // Initialize to creation time
  // ...
};
```

### Code Change - Append Mode
```typescript
// When appending to existing clip (ClipMasterScreen.tsx line 530)
updateClip(currentClipId, {
  rawText: existingClip.rawText + ' ' + rawText,
  status: 'formatting',
  lastModifiedAt: Date.now()  // ✅ ADD THIS to track last interaction
});
```

### Sort Change
```typescript
// In ClipHomeScreen.tsx
const filteredClips = clips
  .filter(clip => !clip.parentId)
  .filter(clip =>
    clip.title.toLowerCase().includes(searchQuery.toLowerCase())
  )
  .sort((a, b) =>
    (b.lastModifiedAt || b.createdAt) - (a.lastModifiedAt || a.createdAt)
  );
```

**Pros**:
- ✅ Clean semantics (createdAt = creation, lastModifiedAt = modification)
- ✅ True "last touched = top" behavior
- ✅ Backward compatible (use `|| createdAt` fallback)

**Cons**:
- ⚠️ Requires schema migration
- ⚠️ More implementation time
- ⚠️ Out of scope for MVP

**Recommendation**: Ship MVP with creation-time sort, add this in v2 if users request it.
