## 🐛 Bug #1: Parent Status Not Showing (v8)

### The Problem
- **Symptom**: Offline recording didn't show "Waiting to transcribe" on parent clip
- **Root Cause**: `homeScreenClips` filtered out children before passing to `ClipHomeScreen`
- **Impact**: `getDisplayClip` couldn't derive parent status from children

### The Fix
**File**: `ClipMasterScreen.tsx`

**Removed** (lines ~1580-1586):
```typescript
const homeScreenClips = useMemo(() => {
  return clips.filter(clip => !clip.parentId);  // ❌ Removed children
}, [clips]);
```

**Changed** (line ~1617):
```typescript
// BEFORE:
<ClipHomeScreen clips={homeScreenClips} />  // ❌ Parents only

// AFTER:
<ClipHomeScreen clips={clips} />  // ✅ All clips (parents + children)
```

### Why This Works
- `ClipHomeScreen` already filters for display internally (line 200)
- `getDisplayClip` now has access to children for status derivation
- Same pattern as `ClipRecordScreen` (which was always working)

## 🐛 Bug #2: Spinner Not Spinning (v9)

### The Problem
- **Symptom**: Status showed "Transcribing..." but spinner stayed static
- **Root Cause**: `processAllPendingClips` never set `activeHttpClipId`
- **Impact**: `isActiveRequest` stayed `false`, spinner didn't spin

### The Fix
**File**: `ClipMasterScreen.tsx` (processAllPendingClips function)

**Added** (after line 540):
```typescript
// Set status to 'transcribing'
updateClip(firstClip.id, { status: 'transcribing' });

// ✅ NEW: Track active HTTP request for spinner animation
setActiveTranscriptionParentId(currentParent.id);
setActiveHttpClipId(firstClip.id);

const result = await attemptTranscription(audioBlob, {...});

// ✅ NEW: Clear tracking after completion
setActiveHttpClipId(null);
setActiveTranscriptionParentId(null);
```

### Why This Works
- Spinner logic requires BOTH `status === 'transcribing'` AND `isActiveRequest === true`
- `isActiveRequest` is derived from comparing `activeHttpClipId === child.id`
- Now matches the working manual retry pattern
- Both screens (ClipHomeScreen + ClipRecordScreen) receive correct prop

## 📊 Complete Data Flow (After Both Fixes)

### Offline Recording → Coming Online

```
1. USER RECORDS OFFLINE
   ↓
   useOfflineRecording creates parent + child
   ↓
   Zustand store: [parent, child]

2. CLIPMASTERSCREEN RECEIVES DATA
   ↓
   const clips = useClipStore((state) => state.clips)
   ✅ HAS: [parent, child] (both present)

3. PASS TO CLIPHOMESCREEN (v8 FIX)
   ↓
   <ClipHomeScreen clips={clips} />
   ✅ RECEIVES: [parent, child] (not filtered)
   ↓
   filteredClips = clips.filter(!parentId)  // Filters for DISPLAY
   ✅ DISPLAYS: [parent only]
   ↓
   getDisplayClip(parent, clips)
   ✅ FINDS: children array has [child]
   ↓
   Derives: status = 'pending-child'
   ↓
   UI SHOWS: "Waiting to transcribe" ✅

4. USER COMES ONLINE
   ↓
   useAutoRetry triggers processAllPendingClips

5. AUTO-RETRY STARTS (v9 FIX)
   ↓
   updateClip(child.id, { status: 'transcribing' })
   ✅ Status = 'transcribing'
   ↓
   setActiveTranscriptionParentId(parent.id)
   ✅ activeTranscriptionParentId = 'parent-123'
   ↓
   setActiveHttpClipId(child.id)
   ✅ activeHttpClipId = 'child-456'
   ↓
   await attemptTranscription(audioBlob)
   ✅ HTTP request in progress

6. CLIPHOMESCREEN DERIVES STATE
   ↓
   getDisplayClip finds child with status 'transcribing'
   derivedIsActiveRequest = (
     activeTranscriptionParentId === parent.id &&  // 'parent-123' === 'parent-123' ✅
     children.some(c => c.id === activeHttpClipId) // 'child-456' in children ✅
   )
   ✅ derivedIsActiveRequest = true
   ↓
   UI SHOWS: "Transcribing..." with SPINNING icon ✅

7. CLIPRECORDSCREEN DERIVES STATE
   ↓
   selectedPendingClips maps:
     isActiveRequest: activeHttpClipId === child.id  // 'child-456' === 'child-456' ✅
   ✅ isActiveRequest = true
   ↓
   UI SHOWS: Child with SPINNING icon ✅

8. TRANSCRIPTION COMPLETES
   ↓
   setActiveHttpClipId(null)
   setActiveTranscriptionParentId(null)
   ↓
   updateClip(parent.id, { content: transcription })
   deleteClip(child.id)
   ↓
   UI SHOWS: Parent completed (no status) ✅
```
