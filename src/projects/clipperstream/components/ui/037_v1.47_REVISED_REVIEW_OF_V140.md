### Addition #3: Phase 6.5 - Update `processParentChildren` Signature

**Add AFTER Phase 6.4 (line 1098)**:

```markdown
#### 6.5 Update `processParentChildren` Function Signature (NEW)

**File**: `ClipMasterScreen.tsx` (search for `processParentChildren` function)

**Current Signature**:
```typescript
async function processParentChildren(
  parentId: string,
  children: Clip[]
): Promise<void>
```

**Update To**:
```typescript
async function processParentChildren(
  parentId: string,
  children: Clip[],
  context: 'auto-retry' | 'new-recording',  // NEW: transcription context
  queueId: string                            // NEW: queue ID for circuit breaker
): Promise<void> {
  for (const child of children) {
    try {
      // Load audio from IndexedDB
      const audioBlob = await getAudio(child.audioId);
      
      if (!audioBlob) {
        console.error('[Auto-retry] Failed to load audio for child clip', { clipId: child.id });
        continue;
      }

      // Pass context and queueId to transcribeRecording
      const result = await transcribeRecording(audioBlob, context, queueId);

      // Handle result (same error logic as new recordings)
      if (result.text && result.text.length > 0) {
        // Success - update clip with transcription
        updateClip(child.id, {
          rawText: result.text,
          usingFallback: result.usingFallback || false,
          status: null  // Clear pending status
        });

        // Format in background
        formatTranscriptionInBackground(child.id, result.text, false);
      } else {
        // Transcription failed - clip remains pending
        console.log('[Auto-retry] Transcription failed for child clip', {
          clipId: child.id,
          error: result.error
        });
        // Clip stays as pending-child, will be retried next time
      }

    } catch (error) {
      console.error('[Auto-retry] Error processing child clip', { clipId: child.id, error });
      // Clip stays as pending-child, will be retried next time
    }
  }
}
```

**Why This Matters**:
- Without signature update, TypeScript will throw compile error
- `context` parameter tells circuit breaker this is auto-retry (not new recording)
- `queueId` parameter enables queue-wide circuit breaker (fails once, learns for entire queue)
```
