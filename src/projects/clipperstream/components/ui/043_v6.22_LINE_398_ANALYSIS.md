## Industry Best Practice

### Pattern: Parent-Child Relationships

**Rule**: The "current context" should always be the **container** (parent), not the **item** (child).

**Examples**:
- File system: Current directory is the folder, not a file inside it
- Email: Current mailbox is the inbox, not an individual email
- Shopping cart: Current context is the cart, not a single item

**Applied to our case**:
- `currentClipId` should be the **parent clip** (container)
- Children are **items within** the parent
- Selector finds children by filtering `parentId === currentClipId`

**Analogy**:
```
You're in a folder (parent)
You create File 1 (child)
You don't "enter" File 1 and change your current directory
You stay in the folder (parent)
You create File 2 (child) in the same folder
```

## What is `currentClipId` For?

**Purpose**: Track which parent clip you're currently working with

**Used by**:
1. **Selector** (line 174): Find children of current parent
2. **Recording** (line 823): Know which parent to append to
3. **Display** (line 1634): Show correct pending clips

**Should it be child ID?** ❌ NO
- Selector breaks (filters by child ID, finds nothing)
- Logic gets confused (child trying to have children)

**Should it be parent ID?** ✅ YES
- Selector works (finds all children)
- Logic clear (parent contains children)
- Recording appends correctly
