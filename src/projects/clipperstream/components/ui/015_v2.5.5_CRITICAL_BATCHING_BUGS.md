### useTranscriptionHandler.ts Batching Logic Bugs

**File**: `/Users/ethan/Documents/projects/final-exp/src/projects/clipperstream/hooks/useTranscriptionHandler.ts`

#### BUG #1: `isFirstPendingForClip` Uses Wrong Logic (lines 93-97)

**Current Code**:
```typescript
const isFirstPendingForClip = useCallback((clipId: string): boolean => {
  const allForClip = clips.filter(c => c.id === clipId);  // ❌ WRONG: Filters by CLIP ID
  const hasContent = allForClip.some(c => c.content);
  return !hasContent;  // First if no content exists yet
}, [clips]);
```

**Problem**:
- This filters for clips with `c.id === clipId`
- But each child has a UNIQUE ID!
- It should filter by PARENT ID: `c.parentId === parentId || c.id === parentId`

**Result**: Every clip thinks it's the "first" pending, so batching never happens!

#### BUG #2: `countRemainingPending` Uses Wrong Logic (lines 100-106)

**Current Code**:
```typescript
const countRemainingPending = useCallback((clipId: string): number => {
  return clips.filter(c =>
    c.id === clipId &&  // ❌ WRONG: Filters by CLIP ID
    c.status === 'transcribing' &&
    c.audioId
  ).length;
}, [clips]);
```

**Problem**:
- This counts clips where `c.id === clipId`
- But we need to count ALL children of the PARENT that are still transcribing
- It should be: `(c.parentId === parentId || c.id === parentId) && c.status === 'transcribing'`

**Result**: `remaining` is always 0 or 1, so the "last one" check never works correctly!

#### BUG #3: Batching Writes All Transcriptions to Last Clip ID (lines 254-263)

**Current Code**:
```typescript
// If last one, display all batched
if (remaining === 1) {  // This is the last one
  log.info('All remaining complete - displaying batch', {
    totalBatched: updatedBatch.transcriptions.length
  });

  (async () => {

      await formatTranscriptionInBackground(
        batch.text,
        targetClip!.id,  // ❌ WRONG: Uses the LAST clip's ID for ALL batches!
```

**Problem**:
- The batch array stores: `{ text: string, audioId: string }`
- But it doesn't store the CLIP ID for each transcription!
- So when formatting, it uses `targetClip!.id` (the LAST clip) for ALL transcriptions
- This means all batched transcriptions overwrite the last clip's content!

**Result**: Clips 2, 3 don't get their transcriptions saved - only the last clip gets overwritten multiple times!

```typescript
const countRemainingPending = useCallback((clip: Clip): number => {
  const parentId = clip.parentId || clip.id;

  // Count all children of this parent that are still transcribing
  const remaining = clips.filter(c =>
    (c.parentId === parentId || c.id === parentId) &&  // Same parent

    c.audioId

    clipId: clip.id,

  });
```
