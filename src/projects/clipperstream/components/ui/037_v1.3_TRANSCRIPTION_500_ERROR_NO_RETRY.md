## Problem Summary

**What Happened**:
1. User recorded clip offline → Saved to IndexedDB ✅
2. User went online → Auto-retry detected clip ✅
3. Transcription API returned **500 Internal Server Error**
4. System treated this as **DEFINITIVE FAILURE** - NO RETRY ❌
5. Clip marked as `status: 'failed'`, never retries again

**User's Observation**:
> "I went online, and it failed to transcribe, showed me an error, no speech detected"

**Root Cause**: **ALL server errors (4xx/5xx) are treated as definitive failures** instead of retry-able errors.

### Why This Is Wrong

**Current Logic**:
- ❌ Timeout errors → RETRY
- ❌ Network errors → RETRY
- ❌ **500 errors → DEFINITIVE FAILURE (NO RETRY)**

**Correct Logic Should Be**:
- ✅ Timeout errors → RETRY
- ✅ Network errors → RETRY
- ✅ **500 errors → RETRY** (server-side issues are often transient)
- ❌ **400-level errors → Depends** (some should retry, some shouldn't)

## Why 500 Errors SHOULD Retry

### What 500 Errors Mean

**500 Internal Server Error** means:
- Server encountered an unexpected condition
- Could be temporary (server overload, API rate limit, temporary service outage)
- Could be transient (memory spike, database connection timeout)
- **NOT the user's fault** - the audio was sent successfully

### Examples of Retry-able 500 Errors

1. **Transcription API temporarily down**
   - First attempt: 500 error
   - Retry after 1 min: Success ✅

2. **API rate limit exceeded**
   - First attempt: 500 error (too many requests)
   - Retry after 2 min: Success ✅

3. **Server memory spike**
   - First attempt: 500 error (out of memory)
   - Retry after 4 min: Success ✅

4. **Database connection timeout**
   - First attempt: 500 error (can't connect to DB)
   - Retry after 1 min: Success ✅

## The Fix

### Option 1: Retry ALL 5xx Errors (Recommended)

**Treat 5xx errors the same as network errors - they should retry.**

**Location**: [useClipRecording.ts:392-410](useClipRecording.ts#L392-L410)

**FIND** (Lines 392-410):
```typescript
// Classify error type
const isTimeout = error instanceof Error && error.name === 'AbortError';
const isNetworkError = error instanceof TypeError;
const isServerError = error instanceof Error && error.message.includes('Server error');

// Server errors (4xx/5xx) are DEFINITIVE failures - don't retry
if (isServerError) {
  setTranscriptionError('Server rejected audio');
  setRetryCount(0);
  setIsActiveRequest(false);
  log.error('Server rejection (definitive failure)', {
    errorMessage: error instanceof Error ? error.message : 'Unknown',
    retriesAttempted: retryCount + 1
  });
  return { text: '', error: 'server-error' };
}

// Network errors (timeout, connection failed) should retry
const shouldRetry = isTimeout || isNetworkError;
```

**REPLACE WITH**:
```typescript
// Classify error type
const isTimeout = error instanceof Error && error.name === 'AbortError';
const isNetworkError = error instanceof TypeError;

// Extract status code from error message (e.g., "Server error 500: ...")
const serverErrorMatch = error instanceof Error && error.message.match(/Server error (\d+):/);
const statusCode = serverErrorMatch ? parseInt(serverErrorMatch[1], 10) : null;

// 5xx errors (server-side issues) should RETRY - they're often transient
// 4xx errors (client-side issues) are DEFINITIVE failures
const is5xxError = statusCode !== null && statusCode >= 500 && statusCode < 600;
const is4xxError = statusCode !== null && statusCode >= 400 && statusCode < 500;

// 4xx errors (client validation failed) are DEFINITIVE failures - don't retry
if (is4xxError) {
  setTranscriptionError('Audio validation failed');
  setRetryCount(0);
  setIsActiveRequest(false);
  log.error('Client error (definitive failure)', {
    statusCode,
    errorMessage: error instanceof Error ? error.message : 'Unknown',
    retriesAttempted: retryCount + 1
  });
  return { text: '', error: 'validation-error' };
}

// Network errors, timeouts, AND 5xx errors should retry
const shouldRetry = isTimeout || isNetworkError || is5xxError;
```

### Why This Fix Works

**New Classification**:

| Error Type | Status Code | Should Retry? | Reason |
|------------|-------------|---------------|--------|
| Timeout | AbortError | ✅ YES | Network issue, try again |
| Network | TypeError | ✅ YES | Connection failed, try again |
| **5xx errors** | **500-599** | **✅ YES** | **Server issue, often transient** |
| 4xx errors | 400-499 | ❌ NO | Client validation failed, won't succeed |

**Example Flows**:

**Scenario 1: Temporary Server Issue (500)**
```
Attempt 1: 500 error → Retry after 1 min
Attempt 2: 500 error → Retry after 2 min
Attempt 3: 500 error → Retry after 4 min
Attempt 4: Success ✅ → User gets their transcription
```

**Scenario 2: Bad Audio Format (400)**
```
Attempt 1: 400 error "Invalid audio format" → STOP (won't retry)
Clip marked as failed ❌ → User sees validation error
```
