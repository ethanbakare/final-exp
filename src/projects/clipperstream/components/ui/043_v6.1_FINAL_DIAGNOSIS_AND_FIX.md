## 🏠 Home Screen Issue: CONFIRMED

### The Bug:

**ClipMasterScreen.tsx Lines 1582-1586:**
```typescript
const homeScreenClips = useMemo(() => {
  return clips.filter(clip => !clip.parentId);  // ❌ REMOVES ALL CHILDREN
}, [clips]);
```

**ClipMasterScreen.tsx Line 1617:**
```typescript
<ClipHomeScreen
  clips={homeScreenClips}  // ❌ Passing filtered array (parents only)
```

### Why This Breaks:

**ClipHomeScreen.tsx Line 145** (inside getDisplayClip):
```typescript
const children = allClips.filter(c => c.parentId === clip.id);
// ❌ ALWAYS returns [] because allClips has NO children
```

### The Fix: ✅ **Remove homeScreenClips filtering**

**STEP 1: Delete lines 1580-1586 in ClipMasterScreen.tsx**
```typescript
// ❌ DELETE THIS ENTIRE BLOCK:
const homeScreenClips = useMemo(() => {
  return clips.filter(clip => !clip.parentId);
}, [clips]);
```

**STEP 2: Change line 1617 in ClipMasterScreen.tsx**
```typescript
// BEFORE:
<ClipHomeScreen
  clips={homeScreenClips}  // ❌ Wrong

// AFTER:
<ClipHomeScreen
  clips={clips}  // ✅ Pass ALL clips (parents + children)
```

**Why this works:**
- ClipHomeScreen already filters for display: `clips.filter(clip => !clip.parentId)` (line 246)
- getDisplayClip now has access to children for deriving parent status
- No breaking changes to ClipHomeScreen API
- No performance impact (same filtering, just in child component)
