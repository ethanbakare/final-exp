### Could the type assertion break getNextRecordingNumber?

**Function signature** (clipStorage.ts:274):
```typescript
export function getNextRecordingNumber(clips: Clip[]): string {
  const parentRecordings = clips.filter(c =>
    c.status !== 'pending-child' &&
    !c.parentId
  );

  const recordingNumbers = parentRecordings
    .map(c => {
      const match = c.title.match(/^Recording (\d+)$/);
      return match ? parseInt(match[1], 10) : 0;
    })
```

**Uses from Clip**:
- `c.status` ✅ Both interfaces have this
- `c.parentId` ✅ Both interfaces have this
- `c.title` ✅ Both interfaces have this

**Impact**: NONE - function only uses fields that exist in both types

**Risk**: ZERO - purely type system fix
