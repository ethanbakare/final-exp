**Before**:
```typescript
else if (activeScreen === 'record' && selectedPendingClips.length > 0) {

  setCurrentClipId(selectedPendingClips[0].id);  // ❌ Sets to CHILD ID
```

**After**:
```typescript
else if (activeScreen === 'record' && selectedPendingClips.length > 0) {
  setIsAppendMode(true);
  // ✅ FIXED (Bug #1): Don't set currentClipId to child ID
  // currentClipId must remain the PARENT ID for selectedPendingClips selector to work
  // Line removed: setCurrentClipId(selectedPendingClips[0].id);
  setAppendBaseContent('');
  setTimeout(() => startRecordingHook(), 200);

```

**Why This Fixes It**:
- `selectedPendingClips` selector filters by `parentId === currentClipId`

- By keeping `currentClipId` as the parent ID, all children are correctly found and displayed

### Fix #2: Text Overwriting (Lines ~590-613)

**Problem**: `processAllPendingClips` was using a stale `currentParent` snapshot captured at the start of the loop. Each clip would append to the *original* parent state, not the *latest* accumulated state.

**Before**:
```typescript
// SUCCESS
if (result.text && result.text.length > 0) {
  console.log('[ProcessPending] Success!');

  // Format transcription
  const formattedText = await formatChildTranscription(
    firstClip.id,
    result.text,
    currentParent.formattedText  // ❌ STALE - captured at loop start
  );

  // Update parent
  const updatedContent = {
    rawText: currentParent.rawText  // ❌ STALE
      ? currentParent.rawText + '\n\n' + result.text
      : result.text,
    formattedText: currentParent.formattedText  // ❌ STALE
      ? currentParent.formattedText + ' ' + formattedText
      : formattedText,
    content: currentParent.formattedText  // ❌ STALE
      ? currentParent.formattedText + ' ' + formattedText
      : formattedText,
  };

  updateClip(currentParent.id, updatedContent);
  // ...
}
```

**After**:
```typescript
// SUCCESS
if (result.text && result.text.length > 0) {
  console.log('[ProcessPending] Success!');

  // ✅ FIXED (Bug #2): Fetch FRESH parent state before appending
  // Prevents stale closure - each clip appends to the LATEST accumulated text
  const freshParent = getClipById(currentParent.id);
  if (!freshParent) {
    console.warn('[ProcessPending] Parent was deleted during formatting, deleting orphaned child');
    deleteClip(firstClip.id);
    if (firstClip.audioId) {
      await deleteAudio(firstClip.audioId);
    }
    deleteAudioRetrievalAttempt(firstClip.id);
    continue;
  }

  // Format transcription
  const formattedText = await formatChildTranscription(
    firstClip.id,
    result.text,
    freshParent.formattedText  // ✅ FRESH
  );

  // Update parent with FRESH data
  const updatedContent = {
    rawText: freshParent.rawText  // ✅ FRESH
      ? freshParent.rawText + '\n\n' + result.text
      : result.text,
    formattedText: freshParent.formattedText  // ✅ FRESH
      ? freshParent.formattedText + ' ' + formattedText
      : formattedText,
    content: freshParent.formattedText  // ✅ FRESH
      ? freshParent.formattedText + ' ' + formattedText
      : formattedText,
  };

  updateClip(currentParent.id, updatedContent);
  // ...
}
```

**Why This Fixes It**:
- Before: `currentParent` captured at loop start → Clip 2 appends to *original* parent (no Clip 1 text)
- After: `freshParent` fetched before each append → Clip 2 appends to *latest* parent (includes Clip 1 text)
- Result: Proper text accumulation across all pending clips

## 🔍 Parent Isolation Verification

**User's Critical Question**: "Are we sure clips from different parent files won't spill over?"

**Answer**: ✅ **Absolutely Safe** - Parent isolation is rock-solid.

### Isolation Mechanism (Line 451)

```typescript
function getFirstPendingClipInParent(parent: Clip): Clip | null {
  const { clips: allClips } = useClipStore.getState();
  const children = allClips.filter(c =>
    c.parentId === parent.id && c.status === 'pending-child'  // ← CRITICAL LINE
  );
  children.sort((a, b) => a.createdAt - b.createdAt);
  return children[0] || null;
}
```

**This line ensures**:
- Only clips with `parentId === parent.id` are fetched
- Children from other parents are **completely invisible**
- Each parent file is processed independently
