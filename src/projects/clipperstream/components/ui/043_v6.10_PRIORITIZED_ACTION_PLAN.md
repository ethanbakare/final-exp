## 🔍 ERROR TOAST CLARIFICATION

### Current Implementation (from ClipToast.tsx)

**Default Messages:**
- **ErrorToast:** `"No audio recorded"` (line 298)
- **VpnToast:** `"Switch off VPN for transcription"` (line 365)
- **AudioToast:** `"Audio saved for later"` (line 229)

### When Each Toast Triggers (from ClipMasterScreen.tsx)

**errorToastMessage** state default: `'No audio detected'` (line 228)

**Toast Shown When:**
1. **Line 796:** Recording too small (`size < 100`) or too short (`duration < 1`)
   - Uses default: `'No audio detected'`

2. **Line 846:** Server error (`transcriptionError === 'server-error'`)
   - Uses default: `'No audio detected'`

3. **Line 869:** Validation error (`transcriptionError === 'validation'`)
   - Uses default: `'No audio detected'`

4. **Line 875:** Fallback error (unknown error type)
   - Uses default: `'No audio detected'`

5. **Line 1516:** Transcription failed (various errors)
   - Uses default: `'No audio detected'`

### Current Issue

The `errorToastMessage` is never customized! It's always `'No audio detected'` for ALL error types.

### Recommendation

Keep using the single generic message since:
- ✅ User-friendly (doesn't expose technical details)
- ✅ Consistent UX
- ✅ Matches existing toast pattern

**Alternative:** Customize per error type AFTER testing confirms all errors properly trigger:
```typescript
// Example customization (OPTIONAL - do later):
if (transcriptionError === 'validation') {
  setErrorToastMessage('No speech detected');
} else if (transcriptionError === 'server-error') {
  setErrorToastMessage('Transcription failed');
} else if (transcriptionError === 'network') {
  setErrorToastMessage('Network error');
} else {
  setErrorToastMessage('No audio detected');
}
```
