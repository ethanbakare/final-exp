#### Implementation (Simple)

```typescript
class TranscriptionCircuitBreaker {
  private state: 'CLOSED' | 'OPEN' | 'HALF_OPEN';
  private failureCount: number;

  constructor() {
    // Restore from localStorage on startup
    const saved = localStorage.getItem('circuit-breaker-state');
    if (saved) {
      const data = JSON.parse(saved);
      this.state = data.state;
      this.failureCount = data.failureCount;
      console.log('[CircuitBreaker] Restored state:', data);
    } else {
      this.state = 'CLOSED';
      this.failureCount = 0;
    }
  }

  private saveState() {
    // Save to localStorage after any change
    localStorage.setItem('circuit-breaker-state', JSON.stringify({
      state: this.state,
      failureCount: this.failureCount,
      lastFailureTime: Date.now()
    }));
  }

  recordFailure(error: Error) {
    this.failureCount++;
    if (this.failureCount >= 2) {
      this.state = 'OPEN';
    }
    this.saveState();  // ← Save after change
  }

  recordSuccess() {
    this.state = 'CLOSED';
    this.failureCount = 0;
    this.saveState();  // ← Save after change
  }
}
```

### Q11: Circuit Breaker Transitions (CLOSED → OPEN → HALF_OPEN → CLOSED)

**Your question**: "You mentioned circuit breaker transitions. I don't really know how it works technically speaking."

**Great question!** Let me explain with a real scenario.

#### The Three States

**1. CLOSED** (Normal operation)
```
State: Everything is working
Action: Try Deepgram for every request
```

**2. OPEN** (Known failure)
```
State: Deepgram is failing (VPN, API down, etc.)
Action: Skip Deepgram, use Whisper for every request
```

**3. HALF_OPEN** (Testing recovery)
```
State: Deepgram might be working again (time has passed)
Action: Try Deepgram once to see if it's back. If success → CLOSED. If failure → OPEN.
```

#### Real-World Scenario with Transitions

**Timeline**:

```
10:00 AM - Circuit is CLOSED
  ↓
User records clip (VPN is ON)
  ↓
10:00:10 - Attempt 1: Deepgram fails (DNS)
          → failureCount = 1

  ↓
10:00:20 - Attempt 2: Deepgram fails (DNS)
          → failureCount = 2
          → Circuit transitions to OPEN
          → lastFailureTime = 10:00
  ↓
10:00:30 - Attempt 3: Circuit is OPEN
          → Skip Deepgram
          → Use Whisper directly ✅
  ↓
10:01:00 - (60 seconds pass since failure)
          → Circuit checks: "Has 60s passed since last failure?"
          → YES (60s threshold reached)
          → Circuit transitions to HALF_OPEN
  ↓
10:01:10 - User records new clip
          → Circuit is HALF_OPEN
          → Try Deepgram once (testing if it's back)
          → SUCCESS! (user turned off VPN)
          → Circuit transitions to CLOSED ✅
  ↓
10:02:00 - User records another clip
          → Circuit is CLOSED
          → Use Deepgram normally ✅
```
