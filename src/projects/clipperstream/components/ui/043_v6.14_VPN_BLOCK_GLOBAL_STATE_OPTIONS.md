# VPN Block Global State - Implementation Options

**Date:** 2026-01-09
**Purpose:** Design options for showing "VPN blocked" on ALL clips while only retrying ONE clip

---

## 🎯 REQUIREMENTS

### What User Wants

1. **Visual State:** When VPN is detected, **ALL clips** should show "VPN blocked"
   - ALL pending children in record screen
   - ALL parent clips in home screen
   - Similar to "waiting to transcribe" - a global state

2. **Processing Logic:** Only **ONE clip** should be retried
   - Stick to the SAME clip (don't jump to next)
   - Keep retrying until VPN is turned off
   - Respect sequence - don't skip clips

### Current Problem

**File:** [ClipMasterScreen.tsx:650-658](ClipMasterScreen.tsx#L650-L658)

```typescript
if (result.error === 'dns-block') {
  console.warn('[ProcessPending] VPN detected, waiting 30s');
  updateClip(firstClip.id, {
    status: 'pending-retry',  // ❌ Changes status from 'transcribing' to 'pending-retry'
    lastError: 'dns-block'
  });
  await new Promise(resolve => setTimeout(resolve, 30000));
  continue; // ← Goes back to while loop
}
```

**What happens next:**
- Line 502: `getFirstPendingClipInParent(currentParent)` is called again
- Line 451: Filters for `c.status === 'pending-child'` ONLY
- **Bug:** Previous clip is now 'pending-retry', so it returns NEXT clip instead!

**Result:** ❌ Jumps to next clip instead of retrying same clip

## 💡 OPTION 1: Global VPN Flag (RECOMMENDED)

### Architecture

Add a **global VPN-blocked flag** to Zustand store that affects **display** but not **processing**.

### Why This is Best

✅ **Separation of concerns:** Display state vs processing state
✅ **Single source of truth:** One flag controls all UI
✅ **Easy to understand:** Clear that VPN affects everything
✅ **Industry standard:** Global app state for global conditions (like offline mode)
✅ **Testable:** Can easily test VPN state without complex mocking
✅ **Scalable:** Works for any number of clips

function getFirstPendingClipInParent(parent: Clip): Clip | null {
  const { clips: allClips } = useClipStore.getState();
  const children = allClips.filter(c =>
    c.parentId === parent.id &&
    (c.status === 'pending-child' || c.status === 'vpn-retrying')  // ✅ Include VPN retrying

## 🏆 RECOMMENDATION: OPTION 1

**Why:**
1. **Industry standard:** Global app state for global conditions (like offline mode in Gmail, Dropbox, etc.)
2. **Clear separation:** Display state (VPN flag) vs processing state (clip status)
3. **Single source of truth:** One flag controls all UI consistently
4. **Easy to test:** Can mock `isVpnBlocked` without complex clip hierarchies
5. **Scalable:** Works for 1 clip or 1000 clips
6. **Future-proof:** Can add "VPN blocked since [timestamp]" UI later

**Real-world examples:**
- **Slack:** Global "offline" banner affects all channels
- **Gmail:** Global "no connection" affects all emails
- **Dropbox:** Global "syncing paused" affects all files

**Implementation complexity:**
- Low: Add 2 state fields, 1 action method
- Medium: Update 3 display components
- Total: ~50 lines of code
