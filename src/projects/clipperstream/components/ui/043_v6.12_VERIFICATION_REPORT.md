### FLAW #1: Incomplete Dependency Array (NEW - Introduced by v6.12)

**File:** [ClipMasterScreen.tsx:682](ClipMasterScreen.tsx#L682)
**Severity:** ⚠️ MEDIUM (Potential stale closure bug)

**Issue:**
```typescript
const processAllPendingClips = useCallback(async () => {
  if (isProcessingPending) {  // ← Line 461: Uses value from closure
    // ...
  }

  const retrievalAttempts = audioRetrievalAttempts.get(firstClip.id) || 0;  // ← Line 530: Uses value from closure

  // ...
}, [formatChildTranscription, setIsProcessingPending, setAudioRetrievalAttempt, deleteAudioRetrievalAttempt]);
// ❌ Missing: isProcessingPending, audioRetrievalAttempts
```

**ESLint Warning:**
```
Line 682:6  warning  React Hook useCallback has missing dependencies: 'audioRetrievalAttempts' and 'isProcessingPending'
```

**Why it's a problem:**
- The function reads `isProcessingPending` (line 461) and `audioRetrievalAttempts` (line 530)
- These values come from Zustand selectors (lines 83, 85)
- The dependency array doesn't include them
- Result: **Stale closure** - function might use old values if these change

**Potential impact:**
- Race condition check (line 461) might not work correctly if `isProcessingPending` changes
- Audio retrieval attempt count (line 530) might be stale

**Fix:**
```typescript
}, [formatChildTranscription, setIsProcessingPending, setAudioRetrievalAttempt, deleteAudioRetrievalAttempt, isProcessingPending, audioRetrievalAttempts]);
```

**OR use getState() pattern:**
```typescript
const processAllPendingClips = useCallback(async () => {
  const { isProcessingPending, audioRetrievalAttempts } = useClipStore.getState();  // Fresh values

  if (isProcessingPending) {
    // ...
  }

  // Use fresh audioRetrievalAttempts
}, [formatChildTranscription, setIsProcessingPending, setAudioRetrievalAttempt, deleteAudioRetrievalAttempt]);
```

**Note:** The function already uses `getState()` pattern on line 472 for `clips`, `updateClip`, `deleteClip`, and `getClipById`. Should apply same pattern to `isProcessingPending` and `audioRetrievalAttempts` for consistency.
