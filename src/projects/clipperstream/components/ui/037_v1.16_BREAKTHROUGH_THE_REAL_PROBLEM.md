## Why "fetch failed"?

This is a **network connectivity error** between your Next.js server and Deepgram's API.

### Possible Causes

1. **Network state issue**: Your server thinks it's online but Deepgram is unreachable
2. **API key issue**: Invalid/expired Deepgram API key (but online recording works?)
3. **Firewall/proxy**: Something blocking the Deepgram API call
4. **Timeout**: Deepgram API not responding in time
5. **DNS resolution failure**: Can't resolve Deepgram's domain

3. **Request context**: Something in the retry flow

## Theory: Race Condition

### Hypothesis
When you go "back online":
1. Browser detects network is back ✅
2. Auto-retry triggers immediately ⚡
3. **But the server/network isn't fully ready yet** ❌
4. Fetch to Deepgram fails because network stabilization takes a moment

### Why Online Recording Works
- Recording happens AFTER network is fully stable
- No race condition with network state changes

### Why Offline-to-Online Fails  
- Triggers TOO FAST after network comes back
- Network isn't fully ready to make external API calls
