## Part 1: Fix Blob MIME Type Loss (Storage Layer)

### Why This Approach

IndexedDB loses the Blob.type property during serialization/deserialization (browser bug). Instead of threading MIME type through the entire app, we fix it at the source: the storage layer.

**Benefits**:
- ✅ Centralized fix - all callers benefit automatically
- ✅ No interface changes
- ✅ No data threading
- ✅ Better encapsulation

---

### Step 1.1: Store MIME Type Separately

**Location**: [audioStorage.ts:66-70](audioStorage.ts#L66-L70)

**FIND**:
```typescript
const request = store.add({
  id: audioId,
  blob,
  timestamp: Date.now()
});
```

**REPLACE WITH**:
```typescript
const request = store.add({
  id: audioId,
  blob,
  mimeType: blob.type || 'audio/webm',  // Store MIME type separately
  timestamp: Date.now()
});
```

### Step 1.2: Recreate Blob with MIME Type on Retrieval

**Location**: [audioStorage.ts:103-108](audioStorage.ts#L103-L108)

**FIND**:
```typescript
if (result) {
  log.debug('Audio retrieved from IndexedDB', {
    audioId,
    size: result.blob.size
  });
  resolve(result.blob);
} else {
  log.warn('Audio not found in IndexedDB', { audioId });
  resolve(null);
}
```

**REPLACE WITH**:
```typescript
if (result) {
  // Recreate blob with correct MIME type if lost during IndexedDB round-trip
  // This fixes a browser bug where Blob.type becomes empty string after deserialization
  const mimeType = result.mimeType || 'audio/webm';
  const correctedBlob = result.blob.type === ''
    ? new Blob([result.blob], { type: mimeType })  // Recreate with stored type
    : result.blob;  // Use original if type was preserved

  log.debug('Audio retrieved from IndexedDB', {
    audioId,
    size: correctedBlob.size,
    originalType: result.blob.type,
    storedMimeType: result.mimeType,
    finalType: correctedBlob.type,
    wasRecreated: result.blob.type === ''
  });

  resolve(correctedBlob);
} else {
  log.warn('Audio not found in IndexedDB', { audioId });
  resolve(null);
}
```

## How This Fix Works

### Before Fix (FAILS)

```
1. Record offline → Blob created with type: 'audio/webm;codecs=opus'
2. Store in IndexedDB → { id, blob, timestamp }
3. Retrieve from IndexedDB → blob.type = '' (LOST!)
4. Send to API → FormData creates File with type: ''
5. API receives → mimetype: '' or undefined
6. Deepgram rejects → 500 error ❌
```

### After Fix (WORKS)

```
1. Record offline → Blob created with type: 'audio/webm;codecs=opus'
2. Store in IndexedDB → { id, blob, mimeType: 'audio/webm;codecs=opus', timestamp }
3. Retrieve from IndexedDB → blob.type = '' BUT mimeType = 'audio/webm;codecs=opus'
4. Recreate blob → new Blob([blob], { type: mimeType })
5. Send to API → FormData creates File with type: 'audio/webm;codecs=opus'
6. API receives → mimetype: 'audio/webm;codecs=opus'
7. Deepgram transcribes → Success ✅
```
