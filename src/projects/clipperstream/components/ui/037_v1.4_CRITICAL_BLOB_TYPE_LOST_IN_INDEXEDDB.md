# 037_v1.4 - CRITICAL: Blob MIME Type Lost When Retrieved from IndexedDB

**Date**: January 1, 2026
**Type**: 🚨 CRITICAL BUG - Root Cause Found
**Status**: ❌ BLOCKING - Offline clips fail with 500 error

---

## Executive Summary

**Problem**: Offline-recorded clips fail with 500 error when transcribing after going online

**Root Cause**: Blob's MIME type is lost when stored/retrieved from IndexedDB, causing API to receive invalid audio format

**Evidence**:
- Online recording → transcription works ✅ (blob has type: 'audio/webm;codecs=opus')
- Offline recording → go online → 500 error ❌ (blob type is empty string: '')

## How Blob MIME Type Loss Causes 500 Error

### The Flow: Online Recording (WORKS)

```
1. MediaRecorder creates blob with type: 'audio/webm;codecs=opus'
   → const blob = new Blob(chunks, { type: 'audio/webm;codecs=opus' })

2. Blob sent to API immediately
   → formData.append('audio', blob, 'recording.webm')
   → File object has type: 'audio/webm;codecs=opus'

3. API receives file with mimetype
   → typedAudioFile.mimetype = 'audio/webm;codecs=opus' ✅
   → mimeType = 'audio/webm;codecs=opus' ✅

4. Deepgram API transcribes successfully ✅
```

---

### The Flow: Offline Recording (FAILS)

```
1. MediaRecorder creates blob with type: 'audio/webm;codecs=opus'
   → const blob = new Blob(chunks, { type: 'audio/webm;codecs=opus' })

2. Blob stored in IndexedDB
   → storeAudio(blob)
   → IndexedDB stores blob object

3. **CRITICAL**: Blob retrieved from IndexedDB
   → getAudio(audioId) returns blob
   → blob.type = '' (EMPTY STRING!) ❌

4. Empty-type blob sent to API
   → formData.append('audio', blob, 'recording.webm')
   → File object has type: '' (EMPTY!)

5. API receives file with NO mimetype
   → typedAudioFile.mimetype = '' or undefined
   → mimeType = '' or 'audio/webm' (fallback) ⚠️

6. Deepgram API receives invalid/empty MIME type
   → Returns 500 error "Transcription failed" ❌
```

---

## Why IndexedDB Loses Blob Type

### The Technical Issue

**IndexedDB and Blob Serialization**:
- IndexedDB stores Blob objects using structured clone algorithm
- The Blob data (bytes) is preserved ✅
- The Blob.type property SHOULD be preserved, BUT:
  - Some browsers have bugs where blob.type becomes empty string
  - IndexedDB might not fully serialize Blob metadata in all cases
  - The type is often lost when the Blob is deserialized

### Browser-Specific Behavior

Different browsers handle Blob serialization differently:
- **Chrome/Edge**: Sometimes lose Blob.type after IndexedDB round-trip
- **Firefox**: More reliable but not guaranteed
- **Safari**: Known issues with Blob serialization in IndexedDB

## The Fix: Store MIME Type Separately

### Solution Overview

Instead of relying on Blob.type being preserved, store the MIME type as a separate field in IndexedDB.

### Part 1: Update storeAudio to Save Type

**Location**: [audioStorage.ts:59-86](audioStorage.ts#L59-L86)

**FIND** (Lines 66-70):
```typescript
const request = store.add({
  id: audioId,
  blob,
  timestamp: Date.now()
});
```

**REPLACE WITH**:
```typescript
const request = store.add({
  id: audioId,
  blob,
  mimeType: blob.type || 'audio/webm',  // ← Store type separately
  timestamp: Date.now()
});
```

### Part 2: Update getAudio to Return Type

**Location**: [audioStorage.ts:93-120](audioStorage.ts#L93-L120)

**CURRENT SIGNATURE**:
```typescript
export async function getAudio(audioId: string): Promise<Blob | null>
```

**NEW SIGNATURE**:
```typescript
export async function getAudio(audioId: string): Promise<{ blob: Blob; mimeType: string } | null>
```

**FIND** (Lines 103-108):
```typescript
if (result) {
  log.debug('Audio retrieved from IndexedDB', {
    audioId,
    size: result.blob.size
  });
  resolve(result.blob);
}
```

**REPLACE WITH**:
```typescript
if (result) {
  // Reconstruct blob with original MIME type if it was lost
  const mimeType = result.mimeType || result.blob.type || 'audio/webm';
  const blob = result.blob.type === ''
    ? new Blob([result.blob], { type: mimeType })  // ← Recreate with type
    : result.blob;  // ← Use original if type preserved

  log.debug('Audio retrieved from IndexedDB', {
    audioId,
    size: blob.size,
    type: blob.type,
    storedMimeType: result.mimeType,
    wasRecreated: result.blob.type === ''
  });

  resolve({ blob, mimeType });
} else {
  log.warn('Audio not found in IndexedDB', { audioId });
  resolve(null);
}
```
