## Technical Details

### What `formatDuration` Does

Converts seconds (number) to display format (string):

```javascript
formatDuration(11)   → "0:11"
formatDuration(65)   → "1:05"
formatDuration(125)  → "2:05"
formatDuration(601)  → "10:01"
```

**Implementation**:
```typescript
const mins = Math.floor(seconds / 60);  // 11 / 60 = 0
const secs = Math.floor(seconds % 60);  // 11 % 60 = 11
return `${mins}:${secs.toString().padStart(2, '0')}`;  // "0:11"
```

### Why This Was Removed

In 037_v1, I removed `formatDuration` from parameters because I thought:
- ❌ "Zustand selector handles pending clips, so formatDuration not needed"
- **WRONG** - formatDuration is needed to create the initial pending clip object

### Correct Approach

Keep `formatDuration` as a **local utility** inside useOfflineRecording:
- ✅ Self-contained (no external dependency)
- ✅ Simple function (5 lines)
- ✅ Stable (never changes)
- ✅ No need to pass from parent

---

## Why Not Pass from Parent?

**Option A**: Pass from ClipMasterScreen (old approach)

```typescript
// ClipMasterScreen.tsx

const { handleOfflineRecording } = useOfflineRecording({

```

**Option B**: Define locally in hook (NEW - better)
```typescript
// useOfflineRecording.ts
const formatDuration = (seconds: number): string => { ... };  // ← Define here
```

**Why Option B is Better**:
1. ✅ Reduces coupling (hook doesn't depend on parent)
2. ✅ Simpler interface (fewer parameters)
3. ✅ Self-contained (all logic in one file)
4. ✅ No useCallback needed (plain function)
