## HYPOTHESIS: The Blob Recreation Problem

When we do:
```typescript
const blob = new Blob([result.data], { type: mimeType });
```

This creates a NEW Blob object, but:
- The browser may handle it differently than a "native" Blob from MediaRecorder
- FormData.append() may serialize it differently
- The server's multipart parser (formidable) may receive corrupted data
