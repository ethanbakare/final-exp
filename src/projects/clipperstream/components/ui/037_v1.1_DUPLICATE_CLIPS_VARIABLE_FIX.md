## Technical Explanation

### Why Duplicate `const` Fails

```javascript
// JavaScript const rules
const clips = [1, 2, 3];  // First declaration - OK
const clips = [4, 5, 6];  // ❌ SyntaxError: Identifier 'clips' has already been declared

// This applies to ALL const declarations in the same scope
function MyComponent() {
  const clips = useClipStore((state) => state.clips);  // Line 69
  // ... 100 lines later ...
  const clips = useClipStore((state) => state.clips);  // Line 153 - ERROR!
}
```

### Why This Compiles to Error

The error shows Next.js/Webpack trying to parse the file:
```
Module parse failed: Identifier 'clips' has already been declared (154:10)
```

The parser encounters the second `const clips` and throws a syntax error because JavaScript doesn't allow redeclaring `const` variables.
