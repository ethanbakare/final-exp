# 🔍 Text Animation Bug Analysis

**Date**: January 10, 2026  
**Issue**: Text sliding animation triggers incorrectly in two scenarios  
**Status**: 🔴 Root cause identified - needs fix

---

## 🎯 Problem Summary

The text sliding animation (`animate-text-intro-horizontal`) is triggering when it shouldn't:

### Scenario 1: Live Recording + Pending Clip
1. User online → Records live → Text transcribes instantly (animation ✅ correct)
2. User goes offline → Records new clip → Pending clip appears below
3. **BUG**: Existing text from step 1 **re-animates** (slides in again) ❌

### Scenario 2: Navigating to Already-Transcribed File (USER CLARIFIED)
1. User offline → Records clip → Pending clip created
2. User navigates to **home screen** (NOT viewing record screen)
3. User comes online → Pending clip transcribes **in background** (user on home screen)
4. User sees title update on home screen (status: transcribing → null)
5. User **clicks into file** after transcription complete
6. **BUG**: Text **animates** on entry ❌

**What SHOULD happen**: Behave like **any other file with existing text** - just show the text normally, no animation.

**Key insight**: Once transcribed, a file is a file. It doesn't matter if it was transcribed live or in background - when you navigate into it, it's just "a file with existing text".

## 🔍 Root Cause Analysis

### Current Animation Logic (ClipRecordScreen.tsx, Lines 106-114)

```typescript
// Get previous content length for this clip
const prevLength = prevContentLengthRef.current[clipId] || 0;
const currentLength = currentText.length;

// ✅ SIMPLE RULE: Animate ONLY when content first appears (0 → non-zero)

// Update tracking for next render

```

**The Problem**: This uses a **component-level ref** (`prevContentLengthRef`) that:
- ✅ Works correctly for the **first** transcription
- ❌ **Resets when component unmounts** (e.g., navigating away)
- ❌ **Doesn't account for multiple children** in the same parent

**Current bug**: Component only knows about Moment 2, not Moment 1.

## 💡 Final Solution: Single Flag, Set at the Right Time

### The `hasAnimated` Flag Logic

**When text appears, ask: "Is user watching the record screen right now?"**

- **YES (watching)** → Don't set flag (leave `undefined`) → Component animates → Component sets `hasAnimated: true`
- **NO (away)** → Set `hasAnimated: true` immediately → Component skips animation

---

### Implementation in Code

1. **Live recording finishes** (user always watching)

2. **Pending transcribes while viewing**

   - Sets `hasAnimated: true`

3. **Pending transcribes while away** (on home screen)

4. **Navigate to any existing file**
   - File already has `hasAnimated: true` (from earlier)
   - Component: `shouldAnimate = false` ✅
   - No animation ✅
   - **Behaves like any normal file with existing text** ✅
