## The VPN Issue

### Why VPN Caused the Problem

**VPN behavior**:
- Routes network traffic through VPN server
- VPN server has its own DNS resolver
- Some VPN DNS resolvers block certain domains
- `api.deepgram.com` was blocked/unreachable via VPN DNS

### Why Online Worked But Offline Failed

**Online recording** (immediate):
- Network request happens quickly (< 1 second after recording)
- VPN might not have fully engaged yet
- Or DNS cache had the address
- Result: ✅ SUCCESS

**Offline recording** (delayed retry):
- Minutes pass between recording and retry
- VPN DNS cache expired
- Fresh DNS lookup needed
- VPN blocks `api.deepgram.com`
- Result: ❌ ENOTFOUND

### Common VPN Behaviors

**VPNs that often block APIs**:
- Free VPNs (aggressive filtering)
- Corporate VPNs (security policies)
- Privacy-focused VPNs (block tracking domains)

**Why they block**:
- DNS-level filtering
- IP blocklists
- Geographic restrictions
- Security policies

### Option 3: Retry with Exponential Backoff

**Add retry logic for DNS errors**:

```typescript
async function transcribeWithRetry(
  audioBuffer: Buffer,
  mimeType: string,
  apiKey: string,
  maxRetries = 3
): Promise<TranscriptionResult> {
  let lastError;

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await transcribeAudio(audioBuffer, mimeType, apiKey);
    } catch (error) {
      lastError = error;

      // Only retry on DNS errors
      if (error.message.includes('ENOTFOUND')) {
        console.log(`DNS error, retry ${attempt}/${maxRetries}`);

        // Exponential backoff: 1s, 2s, 4s
        if (attempt < maxRetries) {
          await new Promise(resolve => setTimeout(resolve, Math.pow(2, attempt - 1) * 1000));
          continue;
        }
      }

      throw error;
    }
  }

  throw lastError;
}
```

**Pros**:
- ✅ Might work if VPN DNS is intermittent
- ✅ No additional services needed

**Cons**:
- ⚠️ Adds delay (3-7 seconds)
- ⚠️ Might still fail if VPN consistently blocks
